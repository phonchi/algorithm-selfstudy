P00 · GREEDY

貪婪何時安全？兩個要素、一種證法 §15.2

DP 在每一步「枚舉所有選擇取最好」；greedy 直接做眼前看起來最好的那個選擇， 不解其他子問題。要讓這招正確，問題必須同時有：

兩個要素 ① 貪婪選擇性質（greedy-choice property）：存在一個「全域最佳解」包含「局部貪婪選擇」—— 可以先做貪婪選擇再解剩下的子問題，不會錯過最佳解。
② 最佳子結構：做完貪婪選擇後剩下的子問題，其最佳解與貪婪選擇合起來是原問題最佳解（與 DP 共用這條）。

證明①的標準武器是交換論證（exchange argument）： 拿任何一個最佳解，把它「不貪婪」的部分換成貪婪選擇，證明換完不會變差—— 所以存在包含貪婪選擇的最佳解。P01 立刻示範。

DP vs Greedy 判斷順序
先想 DP（枚舉選擇）→ 觀察是否「其實每次都選同一種」→ 若能證 exchange argument，降級成 greedy 拿更好的複雜度。反過來先猜 greedy 很容易踩到 0/1 背包型陷阱。

P02 · HUFFMAN

Huffman coding：反覆合併頻率最小的兩棵樹 §15.3

給每個字元一個變動長度、前綴自由（prefix-free）的 0/1 編碼，讓總位元數最少。 貪婪規則：每次從 min-priority queue 取出頻率最小的兩棵樹合併（頻率相加），重複 n−1 次。 用 CLRS 圖 15.5 的頻率表（a:45、b:13、c:12、d:16、e:9、f:5，單位千次）：

▸按「單步」執行下一次合併。
HUFFMAN(C, n)
1Q = C　// min-PQ（按頻率） 2for i = 1 to n − 1 3 x = EXTRACT-MIN(Q) 4 y = EXTRACT-MIN(Q) 5 z = 新節點：z.freq = x.freq + y.freq 6 z.left = x；z.right = y；INSERT(Q, z) 7return EXTRACT-MIN(Q)　// 樹根
n−1 次合併 × O(lg n) 的 PQ 操作 = O(n lg n)。
std::priority_queue<Node*, std::vector<Node*>, Cmp> Q;for (auto [c, f] : freq) Q.push(new Node{c, f});while (Q.size() > 1) { // n−1 次合併 Node* x = Q.top(); Q.pop(); Node* y = Q.top(); Q.pop(); Q.push(new Node{'*', x->freq + y->freq, x, y});}printCodes(Q.top(), ""); // 左 0 右 1 走到葉

# 改寫自官方 Chapter 15/huffman.py（MIT License）：heapq 平鋪完整版 import heapq freq = {"a": 45, "b": 13, "c": 12, "d": 16, "e": 9, "f": 5} # heap 元素 (頻率, 序號, 節點)；序號避免頻率平手時比較 dict counter = 0 h = [] for ch, fr in sorted(freq.items()): h.append((fr, counter, {"ch": ch})) counter += 1 heapq.heapify(h) while len(h) > 1: # n−1 次合併 f1, _, x = heapq.heappop(h) # 頻率最小 f2, _, y = heapq.heappop(h) # 次小 counter += 1 heapq.heappush(h, (f1 + f2, counter, {"left": x, "right": y})) root = h[0][2] codes = {} def walk(node, code): # 左 0 右 1 走到葉 if "ch" in node: codes[node["ch"]] = code or "0" return walk(node["left"], code + "0") walk(node["right"], code + "1") walk(root, "") print("codes:", dict(sorted(codes.items()))) print("期望編碼長度:", sum(freq[c] * len(codes[c]) for c in freq))

兩條引理撐起整個 Huffman（正確性證明骨架）
引理 15.2（貪婪選擇性質）：設 x、y 是頻率最低的兩個字元， 則存在最佳前綴碼樹，其中 x、y 是深度最大的一對兄弟葉子。
證明（交換論證）：取任一最佳樹 T，設 a、b 是 T 中最深的一對兄弟葉。不妨設 freq(x) ≤ freq(a)、freq(y) ≤ freq(b)。 把 x↔a 對調：成本變化 = (freq(a) − freq(x))·(d(a) − d(x)) ≤ 0——頻率小的往深處換不會變貴； 再把 y↔b 對調同理。得到的樹仍最佳且 x、y 是最深兄弟。∎
引理 15.3（最佳子結構）：把 x、y 換成一個頻率 freq(x)+freq(y) 的新字元 z 得字元集 C′。 若 T′ 是 C′ 的最佳樹，把 T′ 中的葉 z 展開成內部節點（掛 x、y）得到的 T 是 C 的最佳樹。
證明要點：成本關係 B(T) = B(T′) + freq(x) + freq(y)（x、y 比 z 深一層，恰多付一次頻率和）； 若 T 非最佳，存在更好的 T″，由引理 15.2 可讓 x、y 在 T″ 中是兄弟，縮回去得到比 T′ 更好的 C′ 樹——矛盾。∎
兩條合體：每次「合併最小兩個」都安全、剩下的子問題遞迴同理 → HUFFMAN 產生最佳前綴碼（定理 15.4）。

P03 · 背包對照

同一題兩種版本：分數背包貪婪成功、0/1 失敗 §15.2

經典對照組（CLRS 圖 15.2）：W = 50，物品 1（w=10, v=60）、物品 2（w=20, v=100）、物品 3（w=30, v=120）。 貪婪規則「按單位價值 v/w 由高到低拿」：

分數背包（fractional）：可拿部分物品。單位價值 6 > 5 > 4， 拿滿物品 1、2，剩 20 容量拿物品 3 的 2/3 → 60 + 100 + 80 = 240（最佳）。 交換論證成立：最佳解若沒裝滿單位價值最高者，把別的東西換掉一定不虧。

0/1 背包：整件拿或不拿。同樣的貪婪拿物品 1、2 → 160； 但最佳解是物品 2 ＋ 3 = 220。貪婪失敗的原因：拿了物品 1 之後剩餘容量被卡住—— 「不可分割」讓局部最優的選擇擋住全域最優。0/1 版沒有貪婪選擇性質，必須用上一章的 DP（Θ(nW)）。

「能不能只做一個選擇就丟掉其餘可能？」能證 → greedy；不能 → 回 DP。考試遇到新題先找反例，找不到再寫 exchange argument。

P04 · AMORTIZED

攤還分析：貴的操作讓便宜的操作分期付款 Ch 16

問的不是「單次操作最壞多少」，而是「n 次操作的序列總共多少、平攤每次多少」—— 沒有任何機率，是最壞情況保證。三種記帳語言，同一個真相：

示範例：k 位元二進位計數器 INCREMENT。單次最壞翻 k 個位元，但 n 次總共翻幾個？按單步看：

本次翻轉
—
累計翻轉
0
平均/次

▸按「單步」執行 INCREMENT。觀察平均值收斂到 2 以下。
三種方法看同一件事
① aggregate：bit 0 每次翻、bit 1 每 2 次、bit i 每 2ⁱ 次 → 總翻轉 Σ n/2ⁱ < 2n → 每次攤還 O(1)。
② accounting：每次 INCREMENT 收 2 元——1 元付「設成 1」，1 元存在該位元上，未來「清成 0」免費。存款永不透支 ⇒ 攤還成本 2。
③ potential：Φ = 目前 1 的個數。攤還成本 = 實際成本 + ΔΦ = (t+1) + (1−t) = 2（t = 本次清掉的 1 數）。
回收資結課的懸案
vector push_back 加倍搬家：accounting 版每次 push 收 3 元（1 放自己、1 搬自己、1 搬「前輩」）→ 攤還 O(1)。§16.4 的 potential 版：Φ = 2·num − size。
實跑：INCREMENT 與 MULTIPOP 的總帳（src/greedy/counter_multipop.py）

動態表格：三種記帳法同台驗算 §16.4

資結課的懸案正式結案：vector「滿了就加倍搬家」，單次 INSERT 最壞 Θ(n)，為什麼敢說攤還 O(1)？ 按單步看 16 次 INSERT 的實際成本——尖峰在 2 的冪出現、其餘全是 1：

本次實際成本
—
num / size
—
Φ = 2num−size
—
▸按「單步」INSERT。虛線 = 攤還線 3。
位能法完整推導（Φ = 2·num − size）
Φ 永遠 ≥ 0（表格至少半滿：num ≥ size/2），Φ₀ = 0 ✓。
不搬家的 INSERT：實際 1，num+1 → ΔΦ = 2，攤還 = 1 + 2 = 3。
搬家的 INSERT（num = size = m 時）：實際 m + 1； 搬家前 Φ = 2m − m = m，搬家＋插入後 num = m+1、size = 2m → Φ = 2 ⟹ ΔΦ = 2 − m。
攤還 = (m + 1) + (2 − m) = 3——擴張瞬間位能「存款」m 全數放出，恰好付掉搬家費。∎
