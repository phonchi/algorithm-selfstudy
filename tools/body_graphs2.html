P00 · MST

MST 問題與切割性質：所有正確性的源頭 §21.1

連通無向帶權圖找一棵生成樹（碰到所有頂點的樹）使邊權總和最小——佈線、修路、電路板走線都是它。 CLRS 先給一個「泛型方法」：維護一個必是某 MST 子集的邊集 A，每次加一條安全邊（加了仍是某 MST 子集）。 Kruskal 與 Prim 只是「挑安全邊」的兩種策略。

切割性質（定理 21.1）——安全邊怎麼找
把頂點切成兩堆 (S, V−S)，若邊集 A 沒有邊橫跨這個切割（切割尊重 A）， 則橫跨切割的最輕邊（light edge）對 A 是安全的。
證明（交換論證，Ch15 貪婪的老朋友）：設 T 是包含 A 的 MST。若 light edge (u,v) ∉ T， T 中 u→v 的唯一路徑必有另一條邊 (x,y) 橫跨切割；換掉它：T′ = T − (x,y) + (u,v) 仍是生成樹， 且 w(u,v) ≤ w(x,y) ⟹ w(T′) ≤ w(T)，所以 T′ 也是 MST 且包含 A ∪ {(u,v)}。∎
兩種策略、一個定理 Kruskal：A 是森林，每次加「連接兩棵不同樹的最輕邊」——切割取「這棵樹 vs 其他全部」。
Prim：A 是一棵從根長大的樹，每次加「離開這棵樹的最輕邊」——切割取「樹內 vs 樹外」。
兩者的每一步都恰是某個切割的 light edge → 由定理 21.1 全程安全 → 結束時是 MST。

P02 · PRIM

Prim 的正確性：每步都是切割的 light edge §21.2

資結課寫過 Prim：從根開始長一棵樹，priority queue 裡放「樹外頂點連進樹的最便宜邊權 key」。 本章補的是為什麼對：每次 EXTRACT-MIN 取出的 (v, key[v])，正是切割「樹內 vs 樹外」的 light edge——定理 21.1 說它安全。整路都安全，長完就是 MST。

同一張圖、兩棵 MST 圖 21.4 有兩條權重 8 的邊：a-h 與 b-c。我們的實測：Kruskal 收 a-h（排序時 a-h 先到、b-c 到時已成環）， 從 a 出發的 Prim 收 b-c（長到 b 時 b-c 先進樹）。兩棵都是 MST、總權重同為 37—— 平手邊讓 MST 不唯一，呼應 P00 的 quiz。

成本：binary heap 版 O(E lg V)（E 次 DECREASE-KEY 主導）；Fibonacci heap 可到 O(E + V lg V)（補充頁）。

P03 · BELLMAN-FORD

鬆弛與 Bellman-Ford：負權也不怕 §22.1

單源最短路徑的核心動作只有一個：RELAX(u, v, w)——若 d[u] + w(u,v) < d[v] 就更新 d[v] 與 π[v]。 所有 SSSP 演算法都只是「用什麼順序鬆弛」的差別。Bellman-Ford 最暴力也最寬容： |V|−1 輪、每輪鬆弛所有邊，允許負權；再多跑一輪，還能鬆就回報「有負環」。 按單步看 CLRS 圖 22.4（含負權邊）逐輪的 d 值：

▸按「單步」執行一輪（鬆弛全部 10 條邊）。∞ = 尚未觸及。
BELLMAN-FORD(G, w, s)
1INIT：d[s]=0，其餘 ∞ 2for i = 1 to |V| − 1 3 for 每條邊 (u, v)：RELAX(u, v, w) 4for 每條邊 (u, v)　// 第 |V| 輪：驗收 5 if d[v] > d[u] + w(u,v)：return FALSE 6return TRUE　// 無負環，d = δ
成本 O(VE)。比 Dijkstra 慢，換來兩個超能力：負權沒問題、能偵測負環。
for (int round = 1; round <= 4; ++round) // |V|-1 輪 for (const Edge& e : E) if (d[e.u] != INF && d[e.u] + e.w < d[e.v]) d[e.v] = d[e.u] + e.w; // RELAXbool ok = true; // 第 |V| 輪：還能鬆 = 負環for (const Edge& e : E) if (d[e.u] != INF && d[e.u] + e.w < d[e.v]) ok = false;

P03b · DAG 最短路

DAG 最短路徑：拓撲序一趟掃完 §22.2

圖若是 DAG，最短路徑突然變便宜：先拓撲排序，再按拓撲序把每個頂點的出邊各鬆弛一次—— 一趟就好、允許負權、Θ(V + E)，比 Dijkstra 還快。正確性直接來自上一章的路徑鬆弛性質： 任何路徑都是拓撲序的子序列，按序鬆弛保證「路徑上的邊按順序各被鬆過一次」。

DAG-SHORTEST-PATHS(G, w, s)
1拓撲排序 G 的頂點 2INIT：d[s]=0，其餘 ∞ 3for 每個頂點 u（按拓撲序） 4 for v ∈ Adj[u]：RELAX(u, v, w)
招牌應用：PERT 關鍵路徑 工程排程：工序當邊、權重 = 工期，最長路徑 = 整案最短完工時間（關鍵路徑）。 把權重取負號跑 DAG 最短路（或把 RELAX 的 min 換成 max）就得解——負權在這裡是常態，Dijkstra 出局、DAG 法剛好。
// 先 DFS 拓撲排序（order 反轉），再按序鬆弛 std::vector<long long> d(n, INF); d[1] = 0; // 源點 s for (int u : order) { // 按拓撲序 if (d[u] == INF) continue; for (auto [v, w] : adj[u]) // 每邊恰鬆弛一次 if (d[u] + w < d[v]) d[v] = d[u] + w; }

P04 · DIJKSTRA

Dijkstra 的證明：貪婪為什麼能定案 §22.3

非負權時可以貪婪加速：像 Prim 一樣用 min-priority queue，每次取出 d 最小的未定案頂點 u， 宣布 d[u] 定案（塗黑），再鬆弛它的出邊。成本 O((V+E) lg V)，海放 Bellman-Ford 的 O(VE)。

正確性不變量（定理 22.6）
取出 u 的當下 d[u] = δ(s,u)。反證：若有更短路徑 p，它必先穿出「已定案集合 S」——設跨出點為 y（前驅 x ∈ S）。 x 定案時鬆過 (x,y) ⟹ d[y] = δ(s,y)；權重非負 ⟹ δ(s,y) ≤ δ(s,u) < d[u]， 那 queue 該先彈 y 而不是 u，矛盾。∎
⚠ 負權為什麼會壞
證明用了「路徑越走越長」（w ≥ 0）。反例：s→a 權 1、s→b 權 2、b→a 權 −2。 Dijkstra 先定案 a（d=1），之後才發現 s→b→a 只要 0——但 a 已定案不回頭，輸出錯的 1。 負權請回頭找 Bellman-Ford；有負環時「最短」根本無定義（繞一圈更短，繞到 −∞）。
DIJKSTRA(G, w, s)
1INIT：d[s]=0，其餘 ∞；S = ∅；全部進 Q 2while Q 非空 3 u = EXTRACT-MIN(Q)；S = S ∪ {u}　// 定案 4 for v ∈ Adj[u]：RELAX(u, v, w)
和 Prim 幾乎同款——差在比較的量：Prim 比「單邊權 key」，Dijkstra 比「從 s 出發的累積距離 d」。
dist[s] = 0; Q.push({0, s});while (!Q.empty()) { auto [du, u] = Q.top(); Q.pop(); if (du > dist[u]) continue; // 過期項目跳過 for (auto [v, w] : adj[u]) if (dist[u] + w < dist[v]) { dist[v] = dist[u] + w; Q.push({dist[v], v}); }}

P05 · FLOYD-WARSHALL

APSP：Floyd-Warshall，DP 回到圖上 §23.2

要「所有點對」的最短路徑，可以跑 V 次 SSSP；但 Floyd-Warshall 用一個漂亮的 DP 一次全解。 子問題定義是精髓（Ch14 四步驟的第 1 步）：

子問題：中繼點受限的最短路徑 dij(k) = 「中繼頂點只准用 {1, …, k}」時 i→j 的最短路徑權重。
遞迴：dij(k) = min( dij(k−1)，dik(k−1) + dkj(k−1) ) ——「不經過 k」對上「經過 k（進 k 一段＋出 k 一段）」。
k 從 0 疊到 n：三層迴圈、O(V³)、五行寫完。允許負權（不允許負環）。
FLOYD-WARSHALL(W, n)
1D = W　// 直達邊權矩陣，對角 0 2for k = 1 to n 3 for i = 1 to n 4 for j = 1 to n 5 dij = min(dij, dik + dkj)

稀疏圖的另一條路：Johnson 重加權 §23.3

FW 是 Θ(V³)；圖很稀疏（E ≪ V²）時「每個頂點跑一次 Dijkstra」只要 O(VE + V² lg V) 更划算—— 但 Dijkstra 怕負權。Johnson 的絕招：把權重「改造」成非負，而且不改變最短路徑：

重加權引理（引理 23.1）
給每個頂點一個值 h(v)，令 w′(u,v) = w(u,v) + h(u) − h(v)。 任何 u→v 路徑 p 的新長度 = 舊長度 + h(u) − h(v)——中間項望遠鏡式相消， 差額只跟端點有關 ⟹ 最短路徑不變（大家都加同一個常數）。
怎麼挑 h 讓 w′ ≥ 0？加一個超級源點 q（連到所有頂點、權 0），令 h(v) = δ(q, v) （Bellman-Ford 算）：三角不等式 δ(q,v) ≤ δ(q,u) + w(u,v) 整理即 w′(u,v) ≥ 0。∎

P06 · MAX FLOW

最大流概覽：殘餘網路與增廣路徑 Ch 24 概覽

流網路：邊有容量，要從源點 s 灌最多的流到匯點 t（守恆：中間頂點進出相等）。 Ford-Fulkerson 方法三個詞就講完：

FORD-FULKERSON 方法 ① 殘餘網路 Gf：還剩多少容量可推——正向剩 c−f，反向多一條容量 f 的「反悔邊」。
② 增廣路徑：Gf 裡 s→t 的任一條路，沿路推「瓶頸」單位的流。
③ 重複到 Gf 沒有 s→t 路徑為止。用 BFS 找路 = Edmonds-Karp，O(VE²) 保證收斂。
Max-flow min-cut 定理（定理 24.7）
三件事等價：f 是最大流 ⟺ Gf 無增廣路徑 ⟺ 存在切割 (S, T) 其容量 = |f|。 最大流的值恰等於最小切割容量——「瓶頸」的嚴格數學版。下例：切割 ({s,v₁,v₂,v₄}, {v₃,t}) 的容量 12 + 7 + 4 = 23，正是最大流。
▸CLRS 圖 24.2（Lucky Puck）。邊標「流/容量」。按「單步」跑一輪 BFS 增廣。
動畫讀法
▸ 邊標 f/c：目前流量／容量。
▸ 橘色 = 本輪增廣路徑；流量推滿（f = c）的邊變紅。
▸ 第三輪 s→v2→v4→v3→t 中的 v4→v3 用的是該邊剩餘的殘餘容量（7 全推滿）。
▸ 結束時：BFS 從 s 只能到 {s, v1, v2, v4}（塗綠）—— 這個可達集就是最小割的 S 側。
EDMONDS-KARP(G, s, t) // Ford-Fulkerson＋BFS 找路
1所有邊 f = 0；建殘餘網路 Gf 2while true 3 在 Gf 上 BFS(s)：只走殘餘容量 > 0 的邊 4 if t 不可達：break　// 無增廣路徑 → 最大流 5 p = BFS 樹上 s→t 的路徑（最少邊數） 6 cf(p) = min{cf(u,v) : (u,v) ∈ p}　// 瓶頸 7 for (u,v) ∈ p：cf(u,v) −= cf(p)；cf(v,u) += cf(p) 8return 累計推的流量
O(VE²) 的理由（定理 24.10 骨架）：BFS 走最短增廣路徑 ⟹ 每個頂點的 BFS 距離 δf(s,v) 隨增廣單調不減；每次增廣至少讓一條「瓶頸邊」飽和消失， 同一條邊要再成為瓶頸、兩端距離必須先 +2——每條邊至多當 O(V) 次瓶頸 ⟹ 增廣至多 O(VE) 輪、每輪 BFS O(E)。
// Edmonds-Karp：BFS 在殘餘網路找路（cap[u][v] > 0 才能走）int f = 1e9; // 瓶頸 = 路徑上最小殘餘容量for (int v = t; v != s; v = parent[v]) f = std::min(f, cap[parent[v]][v]);for (int v = t; v != s; v = parent[v]) { cap[parent[v]][v] -= f; // 正向扣 cap[v][parent[v]] += f; // 反悔邊加}maxFlow += f;
