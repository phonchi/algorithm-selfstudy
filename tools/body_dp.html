P00 · 框架

什麼問題可以 DP？兩大特徵、四個步驟 §14.1、14.3

DP 不是一個演算法，是一種用空間換掉重複計算的設計方法。適用的問題要有兩個特徵：

兩大特徵 ① 最佳子結構（optimal substructure）：問題的最佳解包含子問題的最佳解——「剪下再貼上」論證：若子解不是最佳，換成更好的子解會得到更好的總解，矛盾。
② 重疊子問題（overlapping subproblems）：naive 遞迴會一再解到同一批子問題（子問題總數是多項式，但遞迴樹是指數）——存起來就賺。
四個步驟（§14.3） ① 刻劃最佳解的結構　② 遞迴地定義最佳解的值（遞迴式！）
③ 計算值——bottom-up 填表或 top-down memoization　④ 由表格回溯建構解本身

步驟②的遞迴式是靈魂：之後每個例子都先問「子問題是什麼？狀態怎麼定義？最後一步在哪裡分歧？」。

剪貼論證的完整模板（以 rod cutting 實地走一遍）
主張：長度 n 的最佳切法中，若第一刀切出長度 i，剩下那段（長 n−i）的切法必是「長度 n−i 的最佳切法」。
① 假設不是：剩下那段的切法收益 r′ < rn−i（不是最佳）。
② 剪下：把這段的切法整個拿掉。
③ 貼上：換成長度 n−i 的最佳切法（收益 rn−i）。
④ 矛盾：新總收益 = pi + rn−i > pi + r′ = 原「最佳」收益——比最佳更好，矛盾。∎
矛盾發生在第 ④ 步：替換合法（子問題彼此獨立）是整個論證的隱藏前提—— 最長簡單路徑正是在「替換合法」這一步壞掉（見下方 quiz）。

子問題圖（§14.3）：把每個子問題當頂點、「x 的轉移用到 y」畫一條 x→y 的邊， DP 的成本 = 頂點數 × 平均出邊數（rod cutting：n+1 個頂點、頂點 j 有 j 條出邊 → Θ(n²)）。 bottom-up 填表順序就是這張圖的反向拓撲序——下一章圖論會再遇到它。

vs 分治
分治的子問題不重疊（merge sort 左右兩半獨立）→ 存表沒好處。DP 的子問題大量共用 → 存表把指數壓成多項式。
memo vs bottom-up
同樣 Θ(子問題數 × 轉移成本)。
▸ memo：好寫、只算用到的格子。
▸ bottom-up：無遞迴開銷、順序明確、好回溯。課本與考試以 bottom-up 為主。

P01b · MATRIX CHAIN

矩陣鏈乘法：括號怎麼加，成本差百倍 §14.2

連乘 A₁A₂…Aₙ（Aᵢ 是 pi−1×pi 矩陣），乘法滿足結合律—— 怎麼加括號不影響答案，但嚴重影響成本：A₁(10×100)、A₂(100×5)、A₃(5×50)： ((A₁A₂)A₃) 要 10·100·5 + 10·5·50 = 7500 次乘法；(A₁(A₂A₃)) 要 100·5·50 + 10·100·50 = 75000 次——差 10 倍。 括號化方式有 Catalan 數 Ω(4ⁿ/n3/2) 種，暴力免談；區間 DP 出場：

遞迴式（§14.2） m[i][j] = 算出 Aᵢ..Aⱼ 乘積的最少乘法數；最後一步＝最後一次相乘把鏈切成兩半（切點 k）：
m[i][j] = mini≤k<j ( m[i][k] + m[k+1][j] + pi−1·pk·pj )，m[i][i] = 0。
Θ(n³)：n² 個區間 × 每區間 O(n) 個切點；s[i][j] 記最佳切點供重建。

P02 · LCS

最長共同子序列：二維狀態的招牌題 §14.4

X = ABCBDAB、Y = BDCABA，找最長的共同子序列（可跳字、不可換序）。 狀態 c[i][j] = X 前 i 個字與 Y 前 j 個字的 LCS 長度，看「最後一個字」分歧：

遞迴式（定理 14.1） c[i][j] = 0（i 或 j 為 0）；
c[i][j] = c[i−1][j−1] + 1（xᵢ = yⱼ，↖ 配對）；
c[i][j] = max(c[i−1][j], c[i][j−1])（xᵢ ≠ yⱼ，↑ 或 ← 取大）。
▸按「單步」逐格填表（row-major）。↖ = 配對成功。

0/1 背包：狀態 = (前 i 件, 容量 c) 練習 14.4 系列

P04 · OPTIMAL BST

optimal BST：區間型 DP §14.5

n 個鍵、已知每個鍵被查的機率 pᵢ（與查不到落在間隙的機率 qᵢ），建一棵期望搜尋成本最低的 BST。 高頻鍵不一定當根——重點是整棵樹的期望深度加權和。狀態是鍵的連續區間：

遞迴式（§14.5） e[i][j] = 區間 kᵢ..kⱼ 的最小期望成本 = mini≤r≤j ( e[i][r−1] + e[r+1][j] + w(i,j) )，
其中 w(i,j) = Σp + Σq 是「整個區間深度 +1」的代價。 嘗試每個 r 當根 → Θ(n³)（n² 個區間 × 每區間 O(n) 個根）。

與 rod cutting（一維）、LCS（二維前綴）不同，這是區間 DP——和 P01b 的 matrix chain 同型： 按「區間長度由小到大」填表、枚舉「誰當根」（matrix chain 是「哪裡切」）。完整實作與實跑：

OPTIMAL-BST 完整實作（src/dp/optimal_bst.py，CLRS 圖 14.9 機率表）
# 改寫自官方 Chapter 14/optimal_BST.py（MIT License）：e/w/root 三表全文 def optimal_bst(p, q, n): e = [[0.0] * (n + 1) for _ in range(n + 2)] w = [[0.0] * (n + 1) for _ in range(n + 2)] root = [[0] * (n + 1) for _ in range(n + 1)] for i in range(1, n + 2): e[i][i - 1] = q[i - 1] # 空區間 = 只有間隙 qᵢ₋₁ w[i][i - 1] = q[i - 1] for length in range(1, n + 1): # 區間長度由小到大 for i in range(1, n - length + 2): j = i + length - 1 e[i][j] = float("inf") w[i][j] = w[i][j - 1] + p[j] + q[j] # 整區間「深度+1」代價 for r in range(i, j + 1): # 枚舉根 t = e[i][r - 1] + e[r + 1][j] + w[i][j] if t < e[i][j]: e[i][j] = t root[i][j] = r return e, root p = [0, 0.15, 0.10, 0.05, 0.10, 0.20] q = [0.05, 0.10, 0.05, 0.05, 0.05, 0.10] e, root = optimal_bst(p, q, 5)
執行輸出
期望搜尋成本 e[1][5] = 2.75
整棵樹的根 root[1][5] = k2

k₂ 當根（機率只有 0.10）——不是頻率最高的 k₅（0.20）！最佳 BST 顧的是「整棵樹的加權深度」， 貪婪選高頻當根會失敗（練習 14.5-3 的精神），這正是需要 DP 而不能貪婪的原因——下一章開場的伏筆。
