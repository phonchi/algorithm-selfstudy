P00 · P vs NP

P、NP 與驗證器：解得快 vs 對答案快 §34.1–34.2

先把問題統一成決策問題（答 yes/no；最佳化問題都能加門檻 k 變決策版）。兩個類別：

兩個類別 P：存在多項式時間演算法「解出」答案的問題——排序、最短路徑、MST 全都在這。
NP：給一張證書（certificate）就能在多項式時間「驗證」yes 答案的問題。 例：TSP 的證書是一條導覽，驗證只要加總邊權比門檻；3-CNF 的證書是一組賦值，掃一遍公式就好。
顯然 P ⊆ NP（會解就會驗）；P ≟ NP——「會驗是否代表會解」——是計算機科學最著名的未解問題。
驗證器的正式定義（§34.2）
語言 L 的驗證器是一個兩輸入演算法 A(x, y)：x 是問題實例、y 是證書（certificate）。 A 在多項式時間內回答 0/1，且 L = { x：存在 y、|y| = O(|x|^c)，使 A(x, y) = 1 }。
兩個要件缺一不可：① 證書長度多項式（不能塞一份指數大的小抄）；② 驗證時間多項式。 NP = 有多項式時間驗證器的語言全體。注意不對稱性：定義只管 yes 實例—— 「x 不在 L」不需要短證書（那是 co-NP 的事）。

「NP」是 nondeterministic polynomial 的縮寫：非決定性機器猜一張證書、再多項式驗證—— 不是「non-polynomial」，這是最常見的誤讀。右邊的程式示範「驗證」有多便宜： 給定賦值，檢查 3-CNF 只要線性掃描。難的是在 2ⁿ 種賦值裡找到它。

⚠ 直覺校正
NP 問題不是「無解」也不是「必然要指數時間」——沒有人證明過下界；只是三十年來最聰明的人都沒找到多項式解法。 NPC 理論給的是證據：你的問題和上千個著名難題同生共死。

P01 · REDUCTION

多項式時間化約：難度的搬運工 §34.3

A ≤p B（A 化約到 B）：存在多項式時間函數 f，把 A 的每個實例 x 轉成 B 的實例 f(x)， 且 x 是 yes ⟺ f(x) 是 yes。意義：有 B 的快演算法，就能解 A——「B 至少跟 A 一樣難」。

方向感（考試最容易反）
要證明你的新問題 B 很難：拿一個已知 NPC 的問題 A，做 A ≤p B ——把「公認的難題」偽裝成你的問題的特例。
寫反（B ≤p A）只證明「B 不比 A 難」，什麼難度都沒建立。 口訣：難的在左邊、新的在右邊；箭頭指向你要抹黑的對象。

化約還要證兩個方向的正確性：yes ⟹ yes（構造證書）與 yes ⟸ yes（從 f(x) 的解還原 x 的解）——少一半就不算完成。

P02 · NP-COMPLETE

NP-complete：NP 裡最難的一群 §34.4–34.5

B 是 NP-complete ⟺ ① B ∈ NP，且 ② NP 裡每個問題都 ≤p B（此條件單獨叫 NP-hard）。 第一張骨牌是 Cook–Levin 定理：CIRCUIT-SAT 是 NPC——證法是「任何多項式驗證器都能編成電路」。 之後不用再苦證「每個 NP 問題」：化約有遞移性，從家族樹上任何一個成員接出去就好。

為什麼一個倒、全部倒 若任何一個 NPC 問題有多項式演算法：NP 中每個問題都 ≤p 它 → 全部變多項式 → P = NP。 反過來，只要證明任何一個 NPC 問題沒有多項式解法，就證明了 P ≠ NP。這就是「同生共死」。

P03 · 化約實例

五分鐘化約：CLIQUE ⟷ VERTEX-COVER §34.5.1–34.5.2

家族樹上「CLIQUE → VERTEX-COVER」這步是最漂亮的入門化約，只要一個補圖觀察：

G 有 k-clique ⟺ 補圖 Ḡ 有大小 |V|−k 的 vertex cover
設 V′ ⊆ V 是 G 的 k-clique。看補圖 Ḡ 的任一條邊 (u,v)：它在 G 裡不存在 ⟹ u、v 不會同時在 clique 裡 ⟹ 至少一端落在 V − V′。 所以 V − V′（大小 |V|−k）蓋住 Ḡ 的每條邊——是 vertex cover。 反方向同理（cover 的補集在 G 裡兩兩相鄰 = clique）。∎
化約函數 f：⟨G, k⟩ ↦ ⟨Ḡ, |V|−k⟩，多項式時間可算——完工。

小例：G 是 5 頂點、其中 {a,b,c,d} 兩兩相連的圖（4-clique），e 只連 a。 補圖 Ḡ 的邊只剩 e 的缺邊（e-b、e-c、e-d）——單顆 {e} 就蓋住全部，正是 |V|−k = 5−4 = 1。

P04 · APPROX-VC

2-近似 vertex cover：認難之後的出路 §35.1

ρ(n)-近似演算法：多項式時間，保證解的成本在最佳解的 ρ(n) 倍以內。 VERTEX-COVER 是 NPC，但下面這個笨到不可思議的演算法保證 ρ = 2： 隨便挑一條還沒被蓋住的邊、兩個端點都收、刪掉被蓋住的邊，重複。按單步看 CLRS 圖 35.1：

▸8 條邊待蓋。按「單步」：挑最前面還活著的邊，收兩端。
APPROX-VERTEX-COVER(G)
1C = ∅；E′ = G.E 2while E′ ≠ ∅ 3 任取 (u, v) ∈ E′ 4 C = C ∪ {u, v}　// 兩端都收！ 5 從 E′ 刪掉所有碰到 u 或 v 的邊 6return C
for (size_t i = 0; i < E.size(); ++i) { if (dead[i]) continue; auto [u, v] = E[i]; // 任取一條活邊 C.insert(u); C.insert(v); // 兩端點都收 for (size_t j = 0; j < E.size(); ++j) // 蓋掉鄰邊 if (touches(E[j], u, v)) dead[j] = true;}

為什麼 ≤ 2·OPT（定理 35.1）
演算法挑中的邊集 A 是一個匹配（兩兩不共端點——挑中後鄰邊全刪）。 任何 cover（包括最佳解 C*）都得為 A 中每條邊付出至少一個端點，且這些端點互不重複 ⟹ |C*| ≥ |A|。而 |C| = 2|A| ⟹ |C| ≤ 2|C*|。∎ 妙處：不知道 OPT 是多少，也能證明離它不超過 2 倍——用「匹配」當 OPT 的下界。

P04b · SET COVER

集合覆蓋：貪婪拿下 ln n 近似 §35.3

元素全集 X 與若干子集 S₁…Sₘ，選最少個子集把 X 蓋滿——排班、測試案例挑選、監視器佈點都是它（NP-hard，VERTEX-COVER 的一般化）。 貪婪規則直觀到不行：每輪選「能蓋住最多未蓋元素」的集合。按單步看 CLRS 圖 35.3：

▸12 個元素、6 個集合。按「單步」執行一輪貪婪選擇。
GREEDY-SET-COVER(X, F)
1U = X；C = ∅ 2while U ≠ ∅ 3 選 S ∈ F 使 |S ∩ U| 最大　// 蓋最多新元素 4 U = U − S；C = C ∪ {S} 5return C
while (!U.empty()) { std::size_t best = 0, bestGain = 0; for (std::size_t i = 0; i < F.size(); ++i) { std::size_t gain = 0; for (int e : F[i].second) gain += U.count(e); if (gain > bestGain) { bestGain = gain; best = i; } } for (int e : F[best].second) U.erase(e); cover.push_back(F[best].first); }

近似比 H(max|S|) ≤ ln n + 1 的證明骨架（定理 35.4）
記帳：貪婪選中 Sᵢ 時，把「1 個集合」的成本平攤給本輪新蓋住的元素—— 第 x 個元素分攤 cx = 1/(本輪新蓋數)。總成本 |C| = Σx∈X cx。
關鍵引理：對最佳解的任何集合 S，Σx∈S cx ≤ H(|S|)（調和數）。 直覺：S 的元素被貪婪逐批蓋掉；當 S 還剩 k 個未蓋時，貪婪選的集合至少蓋 k 個新元素 （否則 S 自己更好、貪婪會選它）⟹ 這批元素每個分攤 ≤ 1/k——加總是 1/|S| + … + 1/2 + 1/1 = H(|S|)。
收尾：|C| = Σx cx ≤ ΣS∈C* Σx∈S cx ≤ |C*|·H(max|S|)。∎
這個 ln n 不是分析不夠力——除非 P = NP，set cover 沒有 (1−ε)ln n 的多項式近似（Dinur–Steurer）：貪婪已是漸近最優。

P05 · APPROX-TSP

metric TSP 的 2-近似：MST＋前序走訪 §35.2

TSP 是 NPC；但若成本滿足三角不等式（繞路不會更近，c(u,w) ≤ c(u,v)+c(v,w)——歐氏距離天然滿足）， 兩步就有 2-近似：① 建 MST；② 前序走訪輸出頂點序。按單步看 CLRS 圖 35.2 的 8 個城市：

▸8 個城市（完全圖、歐氏距離）。按「單步」先長 MST。
APPROX-TSP-TOUR(G, c)
1選任意根 r；T = MST-PRIM(G, c, r) 2H = T 的前序走訪頂點序 3return H（首尾相接成環）

⚠ 一般 TSP：連近似都免談（定理 35.3）
沒有三角不等式時，若存在任何常數 ρ 的多項式近似演算法 ⟹ P = NP。 證法又是化約：把 HAM-CYCLE 的圖嵌進完全圖，原有的邊權 1、缺的邊權 ρ|V|+1—— 近似演算法若回傳 ≤ ρ|V| 的導覽，就等於判定了 HAM-CYCLE。近似難度也能用化約證明。

P06 · FPTAS

子集和：從 NPC 到「精度隨你開」的 FPTAS §34.5.5＋§35.5

SUBSET-SUM（給數字集 S 與目標 t，有沒有子集恰好加到 t？）是 NPC——家族樹上由 3-CNF-SAT 化約而來：

3-CNF-SAT ≤p SUBSET-SUM 的數字戲法（§34.5.5 速寫） 每個變數 xᵢ 造兩個十進位大數 vᵢ、vᵢ′（xᵢ = 1 或 0），每個子句 Cⱼ 造兩個「補數」sⱼ、sⱼ′。 數字的每一位對應一個變數或子句：變數位保證「vᵢ、vᵢ′ 恰選一個」（該位加總 = 1）、 子句位保證「每子句至少一個真文字」（該位加總 = 4，補數最多貢獻 3）。 目標 t = 11…1 44…4。十進位不進位（每位總和 ≤ 6）讓「加法」模擬「邏輯」——化約的想像力範本。

最佳化版（不超過 t 的最大子集和）雖然 NP-hard，卻有要多準有多準的近似：

FPTAS（fully polynomial-time approximation scheme）
近似方案：吃參數 ε，保證 (1+ε) 近似。
PTAS：對每個固定 ε 是多項式——但可能是 n^(2/ε)（ε 縮小、次方爆炸）。
FPTAS：時間對 n 與 1/ε 都是多項式（如子集和的 O(n³/ε)）—— 精度變成可以用時間「購買」的商品，NP-hard 問題的最佳待遇。
TRIM：修剪讓清單保持短
EXACT 版逐項維護「所有可能子集和」清單 L（每輪 L ∪ (L+xᵢ)）——長度指數爆炸。
APPROX 版每輪 TRIM(L, δ)、δ = ε/2n：值相差 δ 倍以內的只留一個代表。 每輪相對誤差 ≤ δ、n 輪累積 ≤ (1+δ)ⁿ ≤ 1+ε（ε ≤ 1 時）； 修剪後相鄰值差 > δ 倍 ⟹ 清單長 O((ln t)/δ) = 多項式。準度換長度，帳算得清清楚楚。∎
std::vector<double> trim(const std::vector<double>& L, double delta) { std::vector<double> out = {L[0]}; double last = L[0]; for (double v : L) if (v > last * (1 + delta)) { // 差距夠大才保留 out.push_back(v); last = v; } return out; }
