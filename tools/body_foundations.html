P00 · 演算法與問題

演算法是「把輸入變成輸出的計算程序」 §1.1

資料結構課問「資料怎麼放」；演算法課問「問題怎麼解、解得多快、能不能證明它對」。 CLRS 給的定義很乾脆：演算法（algorithm）是一個定義明確的計算程序， 接受某些值作為輸入（input），產生某些值作為輸出（output）。

談演算法之前要先把「問題」說清楚。一個計算問題（computational problem）描述的是 輸入與輸出之間「應該滿足」的關係。以貫穿 Ch 2 的排序問題為例：

排序問題（sorting problem） 輸入：n 個數的序列 ⟨a₁, a₂, …, aₙ⟩。
輸出：輸入序列的一個重排 ⟨a′₁, a′₂, …, a′ₙ⟩，使得 a′₁ ≤ a′₂ ≤ … ≤ a′ₙ。

給定一組具體的輸入（例如 ⟨5, 2, 4, 6, 1, 3⟩）就得到這個問題的一個實例（instance）。 演算法正確（correct）的意思是：對每一個實例都會停機（halt）、且輸出正確答案。 「對大部分輸入都對」在這門課不及格——我們要的是證明，而證明的主力工具就是本頁 P03 的 loop invariant。

和資料結構課的關係 你在資結課已經寫過 bubble／selection／insertion／shell／merge／quick 六種排序。 這門課把其中兩個（insertion、merge）重新拿出來，但問的問題不一樣： 不再只是「會動」，而是「為什麼一定對？成本函數長什麼樣？」

演算法是一種「技術」：比硬體還有力的加速器 §1.2

好演算法值多少？CLRS §1.2 的經典對決：讓快 1000 倍的電腦跑爛演算法、 慢電腦跑好演算法，各排 1000 萬個數字：

電腦 A（快）電腦 B（慢 1000 倍）
算力10¹⁰ 指令/秒10⁷ 指令/秒
演算法insertion sort，成本 2n² 條指令merge sort，成本 50 n lg n 條指令
n = 10⁷ 時的指令數2·(10⁷)² = 2×10¹⁴50·10⁷·lg 10⁷ ≈ 1.16×10¹⁰
耗時2×10¹⁴ / 10¹⁰ = 20000 秒 ≈ 5.5 小時1.16×10¹⁰ / 10⁷ ≈ 1163 秒 ≈ 20 分鐘

硬體快 1000 倍、常數還小 25 倍（2 vs 50），照樣被演算法翻盤 17 倍——而且 n 越大差距越懸殊（n 再乘 10，A 要 23 天、B 只要 3.7 小時）。 這就是整門課的立足點：演算法和硬體一樣是「技術」，而且是規模越大越值錢的那種。

P01 · 計算模型

RAM 模型：我們用什麼尺量「快慢」 §2.2

說「這個演算法比較快」之前，得先講好用什麼機器量。CLRS 採用 RAM（random-access machine）模型：一台單處理器的抽象機器，指令一條接一條執行（不平行）。

RAM 模型的三條家規：

① 基本指令花常數時間：算術（＋−×÷、取餘、floor/ceiling）、資料搬移（load/store/copy）、控制（分支、副程式呼叫、return），每條都是 O(1)。
② 整數字組寬度有限：存 n 個東西時索引用 c·lg n 位元的字組表示（c ≥ 1 常數）——不能把整個大陣列塞進一個字組裡作弊。
③ 沒有的東西就是沒有：沒有向量指令、沒有快取階層假設、沒有「一次排序一整列」的魔法指令。

在這個模型下，執行時間 = 執行的基本指令條數。 這就是為什麼資結課用 <chrono> 量到的秒數會隨機器不同， 但「成長速率」的結論永遠一致——我們數的是指令條數對輸入大小 n 的函數。

O(1) 一條指令 RAM
x + y、x % y、⌊x/y⌋✓
A[i] 讀寫、變數複製✓
if、函式呼叫、return✓
2ᵏ 乘除（位移，k ≤ 字組寬）✓
「排序整個 A」✗ Θ(n lg n) 子程序
「掃出 min(A)」✗ Θ(n) 子程序
xʸ（任意大指數）✗ 要小心
為什麼要抽象模型
同一支程式在筆電和伺服器上秒數差 10 倍，但「n 加倍、時間變四倍」這件事在哪台機器都成立。模型讓分析與機器脫鉤。

P02 · INSERTION SORT

insertion sort：像整理手上的撲克牌 §2.1

insertion sort 是漸增法（incremental method）的代表： 維持「左手牌永遠排好」這個狀態，每次從桌上抽一張新牌（key）， 在已排好的牌裡從右往左找位置插進去。 下面的動畫直接對照 pseudocode 行號——按「單步」時，先猜下一步陣列會變成什麼樣。

INSERTION-SORT(A, n)
1for i = 2 to n 2 key = A[i] 3 // Insert A[i] into the sorted subarray A[1:i−1]. 4 j = i − 1 5 while j > 0 and A[j] > key 6 A[j+1] = A[j] 7 j = j − 1 8 A[j+1] = key

void insertionSort(std::vector<int>& A) { for (std::size_t i = 1; i < A.size(); ++i) { int key = A[i]; // 將 key 插入已排序的 A[0..i-1]：把比 key 大的元素右移 int j = static_cast<int>(i) - 1; while (j >= 0 && A[j] > key) { A[j + 1] = A[j]; --j; } A[j + 1] = key; }}
執行輸出（輸入 {5, 2, 4, 6, 1, 3}）
1 2 3 4 5 6
0-based 改寫：外圈從 i = 1 起跑、while 條件變 j ≥ 0。j 用 int 而非 size_t，才能安全地減到 −1。
def insertion_sort(A, n): # Traverse the list or array from index 1 to n-1. for i in range(1, n): key = A[i] # Insert A[i] into the sorted subarray A[0:i]. j = i - 1 while j >= 0 and A[j] > key: A[j + 1] = A[j] j -= 1 A[j + 1] = key
執行輸出（輸入 [5, 2, 4, 6, 1, 3]）

P03 · LOOP INVARIANT

loop invariant：用「數學歸納法」證迴圈是對的 §2.1

怎麼證明 insertion sort 一定對？CLRS 的答案是迴圈不變量（loop invariant）： 一句在迴圈每一輪開始前都成立的敘述。對 insertion sort：

INSERTION-SORT 的 loop invariant 在外圈 for 迴圈每輪開始時（索引為 i），子陣列 A[1:i−1] 由「原本就在 A[1:i−1] 裡的元素」組成， 但已排成遞增順序。

要用 loop invariant 證正確性，必須完成三段式論證——結構和數學歸納法一模一樣：

① INITIALIZATION（初始化）
第一輪開始前（i = 2）：A[1:1] 只有一個元素，單一元素天然有序 ✓。對應歸納法的 base case。
② MAINTENANCE（維持）
假設第 i 輪開始前 invariant 成立。這一輪把 A[i] 右邊比它大的元素逐一右移、插入 key， 結束時 A[1:i] 排好、且元素集合不變 → 下一輪開始前 invariant 仍成立 ✓。對應歸納步驟。
③ TERMINATION（終止）
迴圈結束時 i = n + 1。把 i = n + 1 代入 invariant：A[1:n] 已排序、且是原元素的重排——這正是排序問題要的輸出，證畢。 歸納法沒有這段；迴圈會停，所以最後一定要「收割」invariant。
寫證明的檢查清單
▸ invariant 是「狀態敘述」不是「動作敘述」
▸ 三段缺一不可，termination 常被忘記
▸ invariant 要「夠強」：只說「A[1:i−1] 有序」不夠，還要說「元素是原本那些」——否則把陣列全填 0 也滿足
▸ 之後章節同一招：merge 的 k 迴圈、Dijkstra 的集合 S、紅黑樹性質全是 invariant

P04 · 執行時間

逐行計費：從 Σ 到 Θ(n²) §2.2

RAM 模型下，把每行 pseudocode 的「單次成本 × 執行次數」加總就是總時間 T(n)。 insertion sort 的關鍵在第 5–7 行 while 迴圈：令 tᵢ 為外圈第 i 輪 while 條件被測試的次數，則

T(n) 的形狀 T(n) = c₁n + c₂(n−1) + c₄(n−1) + c₅ Σᵢ₌₂ⁿ tᵢ + c₆ Σᵢ₌₂ⁿ (tᵢ−1) + c₇ Σᵢ₌₂ⁿ (tᵢ−1) + c₈(n−1)
一切取決於 Σ tᵢ——也就是「總共右移了幾次」。

最好情況（best case）：輸入已排序。每輪 while 第一次測試就失敗（tᵢ = 1）， T(n) = an + b 是 n 的線性函數 Θ(n)。

最壞情況（worst case）：輸入嚴格遞減。每個 key 都要一路移到最前面（tᵢ = i）， Σᵢ₌₂ⁿ i = n(n+1)/2 − 1，T(n) = an² + bn + c 是 n 的二次函數 Θ(n²)。

這門課預設報告最壞情況：① 它是對任何輸入的保證上限；② 對很多演算法而言最壞情況並不罕見；③「平均情況」需要假設輸入分布（Ch 5 的機率分析），而且 insertion sort 的平均仍是 Θ(n²)——平均每個 key 要移過一半的前綴。

P05 · MERGE SORT

merge sort：第一次正式見到 divide-and-conquer §2.3

漸增法之外的另一條路：分治法（divide-and-conquer）。 遞迴結構固定三步——Divide 把問題切成子問題、Conquer 遞迴解子問題、 Combine 把子解合併成原問題的解。merge sort 把陣列對半切到只剩單元素（天然有序）， 再用 MERGE 逐層「兩疊已排序的牌合併成一疊」。

MERGE-SORT(A, p, r)
1if p ≥ r　　// 0 或 1 個元素？ 2 return 3q = ⌊(p + r)／2⌋　　// 中點 4MERGE-SORT(A, p, q) 5MERGE-SORT(A, q + 1, r) 6MERGE(A, p, q, r)　　// 合併 A[p:q] 與 A[q+1:r]
閉區間 [p, r]。真正做事的 MERGE 是本章最需要逐行吃透的程序——完整 pseudocode、逐步 trace、 不變量證明與互動動畫都在下一節 P06 · MERGE 程序解剖。
void merge(std::vector<int>& A, int p, int q, int r) { std::vector<int> L(A.begin() + p, A.begin() + q + 1); // A[p..q] std::vector<int> R(A.begin() + q + 1, A.begin() + r + 1); // A[q+1..r] std::size_t i = 0, j = 0; int k = p; while (i < L.size() && j < R.size()) A[k++] = (L[i] <= R[j]) ? L[i++] : R[j++]; // 取兩疊頂端較小者 while (i < L.size()) A[k++] = L[i++]; while (j < R.size()) A[k++] = R[j++];}void mergeSort(std::vector<int>& A, int p, int r) { if (p >= r) return; // 0 或 1 個元素：已排序 int q = (p + r) / 2; mergeSort(A, p, q); // Divide + Conquer 左半 mergeSort(A, q + 1, r); // Conquer 右半 merge(A, p, q, r); // Combine}
執行輸出（輸入 {12, 3, 7, 9, 14, 6, 11, 2}）

# 改寫自官方 Chapter 2/merge_sort.py（MIT License）：0-based、閉區間 [p, r] def merge(A, p, q, r): nL = q - p + 1 # A[p..q] 的長度 nR = r - q # A[q+1..r] 的長度 L = [A[p + i] for i in range(nL)] # 複製左半 R = [A[q + 1 + j] for j in range(nR)] # 複製右半 i = j = 0 k = p while i < nL and j < nR: # 兩疊都還有牌：取頂端較小者 if L[i] <= R[j]: # 「≤」平手取左 → 穩定排序 A[k] = L[i] i += 1 else: A[k] = R[j] j += 1 k += 1 while i < nL: # 左疊剩下的依序倒回 A[k] = L[i] i += 1 k += 1 while j < nR: # 右疊剩下的依序倒回 A[k] = R[j] j += 1 k += 1 def merge_sort(A, p, r): if p >= r: # 0 或 1 個元素：已排序 return q = (p + r) // 2 merge_sort(A, p, q) # Divide + Conquer 左半 merge_sort(A, q + 1, r) # Conquer 右半 merge(A, p, q, r) # Combine A = [12, 3, 7, 9, 14, 6, 11, 2] merge_sort(A, 0, len(A) - 1) print(A)

逐層合併動畫

▸按「單步」看每一次 MERGE 呼叫把哪兩段合併。
成本帳
每一「層」的 MERGE 總工作量都是 Θ(n)；
層數 = lg n + 1。
T(n) = 2T(n/2) + Θ(n) = Θ(n lg n)
這條遞迴式（recurrence）怎麼正式解？下一章的 recursion tree 與 Master Theorem。
代價
MERGE 需要 Θ(n) 額外空間（L、R 暫存陣列）——insertion sort 則是就地（in place）排序。快與省，不可兼得（Ch 6 的 heapsort 兩者都要）。

遞迴帳本：為什麼是 Θ(n lg n)

把 T(n) = 2T(n/2) + Θ(n) 逐層展開（n = 8 為例）。每往下一層，子問題數翻倍、每個子問題的合併成本減半—— 每一層的總帳都恰好是 cn：

遞迴展開階梯（n = 8，c = 每元素合併成本）
層 0： [========= n = 8 =========] 合併成本 c·8 = cn 層 1： [== 4 ==] [== 4 ==] c·4 × 2 = cn 層 2： [2] [2] [2] [2] c·2 × 4 = cn 層 3： [1][1][1][1] [1][1][1][1] 單元素不需合併 層數 = lg n + 1 = 4，每層 cn ⟹ T(n) = cn(lg n + 1) = Θ(n lg n)

「層數 × 每層總帳」這招叫 recursion tree，下一章會正式化；那裡也會處理「切不平均」「子問題大小不同」的一般情形。

P06 · MERGE 程序解剖

MERGE：兩疊排好的牌合成一疊 §2.3.1

merge sort 的所有工作都發生在 MERGE 裡，值得整節解剖。 情境：A[p:q] 與 A[q+1:r] 各自已排序（想像桌上兩疊面朝上的牌、每疊最小的在頂端）， 要把它們合併成一段已排序的 A[p:r]。做法：先把兩段抄到暫存陣列 L、R， 再反覆比較兩疊頂端、把較小者放回 A——每次比較恰好送一張牌回家。

MERGE(A, p, q, r) — CLRS 4e 完整版
1nL = q − p + 1　　// A[p:q] 的長度 2nR = r − q　　// A[q+1:r] 的長度 3建立新陣列 L[0:nL−1] 與 R[0:nR−1] 4for i = 0 to nL − 1　// 抄左半進 L 5 L[i] = A[p + i] 6for j = 0 to nR − 1　// 抄右半進 R 7 R[j] = A[q + j + 1] 8i = 0　// L 中最小的未搬元素 9j = 0　// R 中最小的未搬元素 10k = p　// A 中下一個要填的位置 11while i < nL and j < nR　// 兩疊都非空 12 if L[i] ≤ R[j] 13 A[k] = L[i] 14 i = i + 1 15 else A[k] = R[j] 16 j = j + 1 17 k = k + 1 18while i < nL　// L 還有剩：依序倒回 19 A[k] = L[i] 20 i = i + 1 21 k = k + 1 22while j < nR　// R 還有剩：依序倒回 23 A[k] = R[j] 24 j = j + 1 25 k = k + 1
3e 舊版在 L、R 尾端放 ∞ 哨兵（sentinel）省去空疊檢查；4e 改成第 11 行明確檢查 i < nL、j < nR——兩種寫法等價，4e 版不需要「∞ 存在」的假設。

第 2 步的平手正是「≤」的存在理由：相等元素取左邊的，原本在前面的仍在前面——穩定性一行就決定了。 總共 7 次比較送完 8 個元素：每次比較恰送一個元素回家、最後一疊免費倒回，所以比較至多 n − 1 次、搬移恰 n 次——MERGE 是 Θ(n)。

MERGE 主迴圈（第 11–17 行）的 loop invariant——P03 那套的實戰第二例
敘述：每輪開始時，A[p:k−1] 按序放著 L 與 R 中最小的 k − p 個元素； 且 L[i]、R[j] 分別是各自陣列中最小的未搬元素。
Initialization：k = p 時 A[p:k−1] 是空的（0 個元素），i = j = 0 指向兩疊頂端 ✓。
Maintenance：設 L[i] ≤ R[j]（另一邊對稱）。L[i] 同時 ≤ 兩疊所有未搬元素， 而 A[p:k−1] 裡的又都 ≤ L[i]（它們更早被選走）——把 L[i] 放到 A[k] 後， A[p:k] 恰是最小的 k − p + 1 個且有序；i、k 各加 1 讓敘述對下一輪成立 ✓。
Termination：迴圈停時某疊已空。此時 A[p:k−1] 是最小的 k − p 個且有序， 剩下那疊裡全是更大的元素、且自身有序——第 18–25 行依序倒回後 A[p:r] 完全排序。∎

把兩把工具用在新演算法上

練習對象：selection sort（CLRS 習題 2.2-2）。它和 insertion sort 同為 Θ(n²) 的漸增法， 但「前綴」的意義完全不同——先看完整實作與逐輪輸出，再作答下面兩題。

習題 2.2-2 · selectionSort 完整實作（src/foundations/selection_sort.cpp）
void selectionSort(std::vector<int>& A) { for (std::size_t i = 0; i + 1 < A.size(); ++i) { // 只需 n-1 輪：最後一格自動就位 std::size_t m = i; for (std::size_t j = i + 1; j < A.size(); ++j) if (A[j] < A[m]) m = j; // 線性掃出 A[i..n-1] 的最小值 std::swap(A[i], A[m]); } }
逐輪輸出（輸入 {5, 2, 4, 6, 1, 3}）
1 2 4 6 5 3  （前 1 格已定案）
1 2 4 6 5 3  （前 2 格已定案）
1 2 3 6 5 4  （前 3 格已定案）
1 2 3 4 5 6  （前 4 格已定案）
1 2 3 4 5 6  （前 5 格已定案）

注意輸出中的關鍵字「定案」：selection sort 的前綴放的是全域最小的那批，之後永遠不會再動； insertion sort 的前綴只是「自己內部有序」，之後還會被新 key 插隊。另外它的比較次數固定 n(n−1)/2—— 輸入已排序也不會變快（對照 P04 的 best case 表）；而每輪至多一次 swap，總搬移 O(n)，是它唯一勝過 insertion sort 之處。
