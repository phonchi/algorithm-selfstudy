P00 · 框架

什麼問題可以 DP？兩大特徵、四個步驟 §14.1、14.3

DP 不是一個演算法，是一種用空間換掉重複計算的設計方法。適用的問題要有兩個特徵：

兩大特徵 ① 最佳子結構（optimal substructure）：問題的最佳解包含子問題的最佳解——「剪下再貼上」論證：若子解不是最佳，換成更好的子解會得到更好的總解，矛盾。
② 重疊子問題（overlapping subproblems）：naive 遞迴會一再解到同一批子問題（子問題總數是多項式，但遞迴樹是指數）——存起來就賺。
四個步驟（§14.3） ① 刻劃最佳解的結構　② 遞迴地定義最佳解的值（遞迴式！）
③ 計算值——bottom-up 填表或 top-down memoization　④ 由表格回溯建構解本身

步驟②的遞迴式是靈魂：之後每個例子都先問「子問題是什麼？狀態怎麼定義？最後一步在哪裡分歧？」。

vs 分治
分治的子問題不重疊（merge sort 左右兩半獨立）→ 存表沒好處。DP 的子問題大量共用 → 存表把指數壓成多項式。
memo vs bottom-up
同樣 Θ(子問題數 × 轉移成本)。
▸ memo：好寫、只算用到的格子。
▸ bottom-up：無遞迴開銷、順序明確、好回溯。課本與考試以 bottom-up 為主。

P01 · ROD CUTTING

rod cutting：從 2ⁿ 次呼叫到 n² 格表 §14.1

長度 n 的鋼條、價目表 p[1..n]，怎麼切收益最大？遞迴式（最後一步 = 第一刀切在 i）：

遞迴式 rn = max1≤i≤n ( pi + rn−i )，r₀ = 0。
naive 遞迴恰好呼叫 2ⁿ 次（每個子問題被重複解指數次）； 子問題其實只有 n+1 個 → 存起來，Θ(n²)（每格掃一輪 max）。

P02 · LCS

最長共同子序列：二維狀態的招牌題 §14.4

X = ABCBDAB、Y = BDCABA，找最長的共同子序列（可跳字、不可換序）。 狀態 c[i][j] = X 前 i 個字與 Y 前 j 個字的 LCS 長度，看「最後一個字」分歧：

遞迴式（定理 14.1） c[i][j] = 0（i 或 j 為 0）；
c[i][j] = c[i−1][j−1] + 1（xᵢ = yⱼ，↖ 配對）；
c[i][j] = max(c[i−1][j], c[i][j−1])（xᵢ ≠ yⱼ，↑ 或 ← 取大）。
▸按「單步」逐格填表（row-major）。↖ = 配對成功。
速度
LCS-LENGTH(X, Y, m, n) — 核心

for (i = 1; i <= m; ++i) for (j = 1; j <= n; ++j) c[i][j] = (X[i-1] == Y[j-1]) ? c[i-1][j-1] + 1 : std::max(c[i-1][j], c[i][j-1]);// 重建：從 c[m][n] 反向走，相等就收字、否則往大的走

P03 · 0/1 KNAPSACK

0/1 背包：狀態 = (前 i 件, 容量 c) 練習 14.4 系列

n 件物品（重量 wᵢ、價值 vᵢ）、背包容量 W，每件拿或不拿。最後一步看第 i 件：

遞迴式 dp[i][c] = max( dp[i−1][c]，dp[i−1][c−wᵢ] + vᵢ )（後者需 c ≥ wᵢ）；dp[0][·] = 0。
Θ(nW) ——注意這是擬多項式（pseudo-polynomial）：W 是數值不是輸入長度，W 用二進位表示只佔 lg W 位元（Ch 34 再回來談這件事）。
物品 i1234
重量 wᵢ2345
價值 vᵢ3458
KNAPSACK(w, v, n, W)
1dp[0][c] = 0 for all c 2for i = 1 to n 3 for c = 0 to W 4 dp[i][c] = dp[i−1][c]　// 不拿 5 if c ≥ w[i] 6 dp[i][c] = max(dp[i][c], dp[i−1][c−w[i]] + v[i])　// 拿 7return dp[n][W]
for (int i = 1; i <= n; ++i) for (int c = 0; c <= W; ++c) { dp[i][c] = dp[i-1][c]; // 不拿第 i 件 if (c >= w[i-1]) dp[i][c] = std::max(dp[i][c], dp[i-1][c - w[i-1]] + v[i-1]); // 拿 }// 回溯：dp[i][c] ≠ dp[i-1][c] ⟺ 第 i 件被拿了
執行輸出（W = 9）
W = 9 最大價值 = 13，選物品：3 4
# 自寫簡例（官方 clrsPython 未附 knapsack）def knapsack(w, v, W): n = len(w) dp = [[0] * (W + 1) for _ in range(n + 1)] for i in range(1, n + 1): for c in range(W + 1): dp[i][c] = dp[i-1][c] if c >= w[i-1]: dp[i][c] = max(dp[i][c], dp[i-1][c - w[i-1]] + v[i-1]) return dp[n][W]
輸出
knapsack([2,3,4,5], [3,4,5,8], 9) = 13

P04 · OPTIMAL BST

optimal BST：區間型 DP §14.5

n 個鍵、已知每個鍵被查的機率 pᵢ（與查不到落在間隙的機率 qᵢ），建一棵期望搜尋成本最低的 BST。 高頻鍵不一定當根——重點是整棵樹的期望深度加權和。狀態是鍵的連續區間：

遞迴式（§14.5） e[i][j] = 區間 kᵢ..kⱼ 的最小期望成本 = mini≤r≤j ( e[i][r−1] + e[r+1][j] + w(i,j) )，
其中 w(i,j) = Σp + Σq 是「整個區間深度 +1」的代價。 嘗試每個 r 當根 → Θ(n³)（n² 個區間 × 每區間 O(n) 個根）。

與 rod cutting（一維）、LCS（二維前綴）不同，這是區間 DP：按「區間長度由小到大」填表。 同型的還有 matrix-chain multiplication（§14.2，官方 matrix_chain_order）——期中考最愛的手算題型之一。 用 CLRS 圖 14.9 的機率表驗證（官方實作）：
