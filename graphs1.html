P00 · DISJOINT SET

union-find：三個操作、兩個優化 Ch 19

維護一群不相交集合，支援 MAKE-SET／FIND-SET（回傳代表）／UNION。 森林實作：每個集合一棵樹、根是代表。裸版最壞會退化成鏈；加上兩個優化後脫胎換骨：

兩個優化 ① union by rank：矮樹掛到高樹下（rank 是高度上界），樹高 O(lg n)。
② path compression：FIND-SET 沿途把每個節點直接改指到根——查一次、修一路。
兩者合用：m 次操作總成本 O(m·α(n))，α 是反 Ackermann 函數——宇宙尺寸的 n 也 ≤ 4，「幾乎常數」。（證明用 Ch16 的 potential method——攤還分析的實戰舞台。）
▸CLRS 圖 19.1 的例子：依序 union 建出連通元件。按「單步」。
FIND-SET(x) // 含 path compression
1if x ≠ x.p 2 x.p = FIND-SET(x.p)　// 沿途全改指根 3return x.p
UNION(x, y) // by rank
1rx = FIND-SET(x)；ry = FIND-SET(y) 2矮的掛到高的下面；同高則任選並 rank+1
int findSet(int x) { if (p[x] != x) p[x] = findSet(p[x]); return p[x];}void unite(int x, int y) { int rx = findSet(x), ry = findSet(y); if (rx == ry) return; if (rank_[rx] < rank_[ry]) std::swap(rx, ry); p[ry] = rx; if (rank_[rx] == rank_[ry]) ++rank_[rx];}

P01 · BFS

BFS 深化：為什麼 d[v] 就是最短距離 §20.2

資結課會寫 BFS；本章補上三件武器： ① 白灰黑——白＝未發現、灰＝在 queue 裡（波前）、黑＝鄰居掃完。 ② BFS tree——π 指標串出的樹，樹上路徑就是最短路徑。 ③ 正確性（定理 20.5）：對無權圖，BFS 結束時 d[v] = δ(s, v)（真實最短距離）。

證明骨架
上界：d[v] ≥ δ(s,v)（每次入列 d 都是「某條真實路徑」的長度）。
Queue 不變量：佇列裡的 d 值單調不減、且首尾差 ≤ 1——波一層層擴散、不會跳層。
反證：假設有 v 使 d[v] > δ(s,v)，取 δ 最小的違規者，看它最短路徑上的前驅 u：v 由 u 鬆弛時必得 d[v] = d[u] + 1 = δ(s,v)，矛盾。∎

成本 O(V + E)：每個頂點入列一次、每條邊掃一次——鄰接串列的紅利。

BFS(G, s) — 核心
1s 塗灰、d[s]=0、入列 2while Q 非空 3 u = DEQUEUE(Q) 4 for v ∈ Adj[u]：if v 是白 5 v 塗灰；d[v]=d[u]+1；π[v]=u；入列 6 u 塗黑
std::vector<int> bfs(const Graph& G, int s) { std::vector<int> d(G.n, -1); // -1 = WHITE std::queue<int> Q; d[s] = 0; Q.push(s); while (!Q.empty()) { int u = Q.front(); Q.pop(); for (int v : G.adj[u]) if (d[v] < 0) { d[v] = d[u] + 1; Q.push(v); } } return d;}
執行輸出（CLRS 圖 20.3，源點 s）
BFS 距離：r:1 s:0 t:2 u:3 v:2 w:1 x:2 y:3

P02 · DFS

DFS 時間戳與括號定理：一切證明的座標系 §20.3

DFS 給每個頂點兩個時間戳：發現 u.d（塗灰）與完成 u.f（塗黑）。 按單步看穿衣 DAG 的 DFS 產生的「括號序列」：

▸按「單步」推進 DFS。( = 發現（塗灰）、) = 完成（塗黑）。
括號定理（定理 20.7）
任兩頂點的 [d, f] 區間要嘛互不相交、要嘛完整巢狀——絕不交錯。 v 是 u 的後裔 ⟺ u.d < v.d < v.f < u.f。
邊分類（用顏色判定）
探索邊 (u, v) 時看 v 的顏色：
白 → tree edge
灰 → back edge（指向祖先 = 有環！）
黑 → forward／cross edge
有向圖無環 ⟺ DFS 無 back edge（引理 20.11）。

P03 · TOPOLOGICAL SORT

拓撲排序 = DFS 完成時間的反序 §20.4

資結課的 optional 主題轉正：DAG 的線性排序，使每條邊都由左指右。作法一行話—— 跑 DFS，頂點按完成時間 f 遞減排列（完成先後倒過來）。

為什麼對（定理 20.12）
對任意邊 (u, v)：探索它時 v 不可能是灰的（灰 = 祖先 → back edge → 有環，DAG 排除）。 v 白則成為 u 的後裔 → v.f < u.f；v 黑則 v 早就完成 → v.f < u.f。 兩種情況都有 v.f < u.f → 按 f 遞減排，u 一定排在 v 前面。∎
C++／官方 Python 相同輸出（CLRS 圖 20.7 穿衣 DAG）
topological sort：錶 襪子 襯衫 領帶 內褲 褲子 鞋 皮帶 外套

同一個 DAG 可以有多個合法拓撲序；換 DFS 的起點順序會得到不同但同樣正確的結果。另一個常用做法是 Kahn 演算法（重複移除入度 0 的頂點）——課本習題 20.4-5。

P04 · SCC

強連通元件：兩趟 DFS 的魔術 §20.5

SCC = 極大的「彼此互達」頂點集。把每個 SCC 縮成一點得到的 component graph 必是 DAG。演算法三行：

STRONGLY-CONNECTED-COMPONENTS(G) ① 對 G 跑 DFS，記下每個頂點的完成時間 f。
② 建轉置圖 Gᵀ（所有邊反向）。
③ 對 Gᵀ 跑 DFS，但主迴圈按 f 遞減的順序取起點——每棵 DFS 樹恰是一個 SCC。

直覺：f 最大的頂點位在 component graph 的「源頭」SCC；在 Gᵀ 裡從它出發，只能走回自己的 SCC（其他方向的邊都被反向堵住）——一棵樹收割一個元件，收割完換下一個源頭。

SCC(G)（CLRS 圖 20.9 例：a→b→c→d、b→e、e→a、…）
1DFS(G) 記 f 2建 Gᵀ 3DFS(Gᵀ)，主迴圈按 f 遞減選起點 4每棵第二趟 DFS 樹 = 一個 SCC
void dfs1(int u) { // 第一趟：記完成順序 visited[u] = 1; for (int v : adj[u]) if (!visited[v]) dfs1(v); order.push_back(u);}void dfs2(int u, int c) { // 第二趟：在 Gᵀ 收割 comp[u] = c; for (int v : radj[u]) if (comp[v] < 0) dfs2(v, c);}// 主程式：dfs1 全圖 → 反向走 order → dfs2
