P00 · HEAPSORT

heapsort：你已經會 heap，現在讓它排序 Ch 6

資結課的 binary heap 三件事複習：陣列存完全二元樹（0-based：left = 2i+1、right = 2i+2）、 max-heap 性質（父 ≥ 子）、MAX-HEAPIFY 往下沉修復（O(lg n)）。heapsort 只加兩個觀察：

① BUILD-MAX-HEAP：從最後一個內部節點（⌊n/2⌋−1）往前逐一 heapify——葉子天然是 heap，不用管。 精算總成本不是 O(n lg n) 而是 O(n)：大多數節點靠近底層、下沉距離很短（Σ h·n/2^(h+1) = O(n)）。
② 排序主迴圈：堆頂（最大值）和尾端互換 → heap 縮小一格 → heapify 修復。重複 n−1 次， 陣列尾端長出一條「已排序尾巴」。Θ(n lg n)、就地、不穩定——merge sort 的速度＋insertion sort 的空間。

P01 · PARTITION

quicksort 的引擎：Lomuto partition §7.1

quicksort 的所有工作都在 PARTITION 裡：選 A[r] 當 pivot x， 一趟掃描把陣列整理成「≤ x｜> x｜x」，最後把 pivot 換到中間正確的最終位置。 兩個指標的 invariant：A[p..i] ≤ x、A[i+1..j−1] > x。動畫用 CLRS 圖 7.1 的 ⟨2,8,7,1,3,5,6,4⟩：

▸pivot = 4（紅框）。按「單步」看 j 掃描、i 維持「≤ pivot 區」右界。

1x = A[r]　// pivot 2i = p − 1 3for j = p to r − 1 4 if A[j] ≤ x 5 i = i + 1 6 exchange A[i] with A[j] 7exchange A[i+1] with A[r] 8return i + 1

1if p < r 2 q = PARTITION(A, p, r)　// 隨機化版：先把隨機位置換到 A[r] 3 QUICKSORT(A, p, q − 1)　// pivot 已就定位，不再參與 4 QUICKSORT(A, q + 1, r)
RANDOMIZED-PARTITION 只多一行：i = RANDOM(p, r)；exchange A[r] ↔ A[i]。就這一行把最壞情況從「特定輸入必發生」變成「任何輸入都只有極低機率」。
int partition(std::vector<int>& A, int p, int r) { int x = A[r]; // pivot 取最後一個元素 int i = p - 1; // 「≤ x 區」的右邊界 for (int j = p; j < r; ++j) if (A[j] <= x) std::swap(A[++i], A[j]); std::swap(A[i + 1], A[r]); // pivot 歸位 return i + 1;}void randomizedQuicksort(std::vector<int>& A, int p, int r, std::mt19937& rng) { if (p >= r) return; std::uniform_int_distribution<int> d(p, r); std::swap(A[d(rng)], A[r]); // 隨機挑 pivot 換到尾端 int q = partition(A, p, r); randomizedQuicksort(A, p, q - 1, rng); randomizedQuicksort(A, q + 1, r, rng);}

P02 · QUICKSORT 分析

最壞 Θ(n²)、期望 Θ(n lg n)：差別全在切分平衡 §7.2、7.4

最壞情況：每次 pivot 都是最大或最小（例如已排序輸入＋固定取尾端），切分成 0 : n−1， T(n) = T(n−1) + Θ(n) = Θ(n²)——遞迴樹退化成一條鏈。

只要「成比例」就贏：即使每次都切 9 : 1，遞迴樹深度仍是 log₁₀⸝₉ n = Θ(lg n)、每層 O(n) → O(n lg n)。切分不必完美，常數比例就夠——這是 P02 recursion tree 那題不平衡樹的直接應用。

隨機化的期望分析（§7.4.2 的漂亮論證）：令 z₁<…<zₙ 為排序後元素， zᵢ 與 zⱼ要嘛比較一次、要嘛永不比較——只有當 zᵢ 或 zⱼ 是「區間 Zᵢⱼ = {zᵢ,…,zⱼ} 中第一個被選到的 pivot」才會比較：

indicator RV 再上場 Pr{zᵢ 與 zⱼ 比較} = 2／(j − i + 1)。期望總比較數 E[X] = Σᵢ Σⱼ 2/(j−i+1) ≤ Σᵢ O(lg n) = O(n lg n)。 上一章 hiring problem 的工具原封不動搬過來。
為什麼實務最愛 quicksort
▸ 就地（O(lg n) 遞迴堆疊）、常數小、cache 友善。
▸ 隨機化後沒有「天生最壞輸入」。
▸ std::sort 的 introsort = quicksort ＋ 深度超標時切 heapsort ＋ 小區間 insertion sort：三章的知識拼在同一支函式裡。

P03 · 下界

比較排序的天花板：Ω(n lg n) §8.1

merge sort 和 heapsort 都是 Θ(n lg n)——還能更快嗎？如果演算法只透過元素兩兩比較取得順序資訊，答案是不能：

定理 8.1（決策樹論證）
把比較排序的執行過程畫成決策樹（decision tree）：內部節點是「aᵢ ≤ aⱼ？」，葉子是輸出的排列。 正確的演算法必須能產生全部 n! 種排列 → 葉子 ≥ n!。 高度 h 的二元樹葉子 ≤ 2ʰ，所以 2ʰ ≥ n! → h ≥ lg(n!) = Ω(n lg n)（Stirling）。 最壞情況的比較次數 = 根到最深葉的路徑長 ≥ Ω(n lg n)。∎

這是本課程第一個問題本身的下界——不是「還沒想到更快的方法」，而是「這個模型裡不存在更快的方法」。 merge sort／heapsort 因此是漸近最優的比較排序。想更快只有一條路：換模型——不比較，改用元素的「數值本身」當索引，這就是下一節。

P04 · 線性時間排序

counting → radix → bucket：不比較，用「數的」 §8.2–8.4

counting sort：值域 [0, k] 的整數。C[v] 先數「值 = v 有幾個」，前綴和後 C[v] =「≤ v 的個數」—— 這個數字就是 v 該站的位置。從後往前放進輸出陣列 B，相等元素保持原相對順序（穩定）。按單步看放置過程：

A（輸入）
C（累計 ≤ v 個數）
B（輸出）
▸CLRS 圖 8.2 的例子：A = ⟨6,0,2,0,1,3,4,6,1,3,2⟩、k = 6。按「單步」從尾端開始放。
COUNTING-SORT(A, n, k)
1C[0:k] 全設 0 2for j = 1 to n：C[A[j]] += 1 3for v = 1 to k：C[v] += C[v−1] 4for j = n downto 1 5 B[C[A[j]]] = A[j] 6 C[A[j]] −= 1
std::vector<int> countingSort( const std::vector<int>& A, int k) { std::vector<int> C(k + 1, 0), B(A.size()); for (int a : A) ++C[a]; for (int v = 1; v <= k; ++v) C[v] += C[v - 1]; for (int i = A.size() - 1; i >= 0; --i) { B[C[A[i]] - 1] = A[i]; // 從後往前 → 穩定 --C[A[i]]; } return B;}
輸出
0 0 1 1 2 2 3 3 4 6 6

注意官方版的 k 是「值域大小」（值 0..k−1），課本是「最大值 k」——差一。

radix sort（§8.3）：d 位數的整數，從最低位開始逐位用「穩定排序」（通常就是 counting sort）排 d 輪 → Θ(d(n + k))。穩定性是命脈：排高位時，低位已排好的相對順序必須被保留。 bucket sort（§8.4）：輸入均勻分布在 [0,1) 時，切 n 個桶、各桶 insertion sort，期望 Θ(n)——又是機率分析。

P05 · ORDER STATISTICS

randomized select：找第 i 小，期望 Θ(n) §9.1–9.2

找中位數（或第 i 小）要排序嗎？不用。RANDOMIZED-SELECT 借 quicksort 的 partition， 但每次只遞迴一邊：pivot 落點 q 把問題切成兩半，答案只會在其中一半。 期望分析：pivot 有一半機率落在「中間 50%」，每兩輪期望把問題砍到 3/4 → 幾何級數 n + 3n/4 + 9n/16 + … = Θ(n)。

對照：quicksort 兩邊都遞迴 → Θ(n lg n)；select 只走一邊 → Θ(n)。 同一個 partition，遞迴結構決定級別。另外 §9.3 的 SELECT（中位數之中位數、5 個一組）把「期望線性」升級成最壞情況線性——保證 pivot 至少甩掉 30% 的元素，代價是常數較大，實務反而少用。
RANDOMIZED-SELECT(A, p, r, i)　// 第 i 小
1if p == r：return A[p] 2q = RANDOMIZED-PARTITION(A, p, r) 3k = q − p + 1　// pivot 在本段排第 k 小 4if i == k：return A[q] 5elseif i < k：return RANDOMIZED-SELECT(A, p, q−1, i) 6else：return RANDOMIZED-SELECT(A, q+1, r, i−k)
第 6 行的 i − k：進入右半段後「目標排名」要扣掉左半段（含 pivot）的 k 個元素。
int randomizedSelect(std::vector<int>& A, int p, int r, int i, std::mt19937& rng) { if (p == r) return A[p]; std::uniform_int_distribution<int> d(p, r); std::swap(A[d(rng)], A[r]); int q = partition(A, p, r); int k = q - p + 1; if (i == k) return A[q]; if (i < k) return randomizedSelect(A, p, q - 1, i, rng); return randomizedSelect(A, q + 1, r, i - k, rng);}
