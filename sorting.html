P00 · HEAPSORT

heapsort：你已經會 heap，現在讓它排序 Ch 6

資結課的 binary heap 三件事複習：陣列存完全二元樹（0-based：left = 2i+1、right = 2i+2）、 max-heap 性質（父 ≥ 子）、MAX-HEAPIFY 往下沉修復（O(lg n)）。heapsort 只加兩個觀察：

① BUILD-MAX-HEAP：從最後一個內部節點（⌊n/2⌋−1）往前逐一 heapify——葉子天然是 heap，不用管。 精算總成本不是 O(n lg n) 而是 O(n)：大多數節點靠近底層、下沉距離很短（Σ h·n/2^(h+1) = O(n)）。
② 排序主迴圈：堆頂（最大值）和尾端互換 → heap 縮小一格 → heapify 修復。重複 n−1 次， 陣列尾端長出一條「已排序尾巴」。Θ(n lg n)、就地、不穩定——merge sort 的速度＋insertion sort 的空間。

BUILD-MAX-HEAP 為什麼是 O(n) 而不是 O(n lg n)

緊分析：把成本按「高度」記帳
粗算：n/2 次 heapify、每次 O(lg n) → O(n lg n)。沒錯但不緊——heapify 的真實成本是 O(h)（h = 該節點的高度）， 而大多數節點高度很小。
n 個元素的 heap 中，高度為 h 的節點至多 ⌈n/2^(h+1)⌉ 個（一半是葉子 h=0、四分之一是 h=1、…）。總成本：
Σh=0⌊lg n⌋ ⌈n/2^(h+1)⌉·O(h) = O( n·Σh=0∞ h/2^h )。
關鍵級數：Σ h/2^h = 2（把 Σ x^h = 1/(1−x) 微分再乘 x，代 x = 1/2：x/(1−x)² = 2）。
所以總成本 = O(2n) = O(n)。∎　直覺：便宜的節點（矮的）佔絕大多數、貴的節點（高的）寥寥無幾——加權平均下來每個節點攤不到常數。
QUIZ P00 · BUILD-MAX-HEAP

BUILD-MAX-HEAP 為什麼從 ⌊n/2⌋ 往 1 倒著跑（0-based：n/2−1 → 0），而不是從 1 順著跑？

(A) heapify 的前提是「兩個子樹已是 heap」，必須由下而上
(B) 倒著跑比較快，順著跑會變 O(n²)
(C) 因為葉子不用處理
P00b · PRIORITY QUEUE

優先佇列：heap 的第二個人生 §6.5

heap 不只是排序引擎——它是最大優先佇列（max-priority queue）的標準實作： 維護一個集合，隨時要能「看最大、取最大、調高某元素的優先權、插入新元素」。 OS 的工作排程、事件驅動模擬、之後 Prim／Dijkstra 的 min 版本，全是這組介面：

INCREASE-KEY增鍵後沿父鏈往上浮（heapify 是往下沉）O(lg n)

P01 · PARTITION

quicksort 的引擎：Lomuto partition §7.1

quicksort 的所有工作都在 PARTITION 裡：選 A[r] 當 pivot x， 一趟掃描把陣列整理成「≤ x｜> x｜x」，最後把 pivot 換到中間正確的最終位置。 兩個指標的 invariant：A[p..i] ≤ x、A[i+1..j−1] > x。動畫用 CLRS 圖 7.1 的 ⟨2,8,7,1,3,5,6,4⟩：

▸pivot = 4（紅框）。按「單步」看 j 掃描、i 維持「≤ pivot 區」右界。

QUICKSORT(A, p, r)　／　RANDOMIZED-PARTITION
1if p < r 2 q = PARTITION(A, p, r)　// 隨機化版：先把隨機位置換到 A[r] 3 QUICKSORT(A, p, q − 1)　// pivot 已就定位，不再參與 4 QUICKSORT(A, q + 1, r)
RANDOMIZED-PARTITION 只多一行：i = RANDOM(p, r)；exchange A[r] ↔ A[i]。就這一行把最壞情況從「特定輸入必發生」變成「任何輸入都只有極低機率」。
int partition(std::vector<int>& A, int p, int r) { int x = A[r]; // pivot 取最後一個元素 int i = p - 1; // 「≤ x 區」的右邊界 for (int j = p; j < r; ++j) if (A[j] <= x) std::swap(A[++i], A[j]); std::swap(A[i + 1], A[r]); // pivot 歸位 return i + 1;}void randomizedQuicksort(std::vector<int>& A, int p, int r, std::mt19937& rng) { if (p >= r) return; std::uniform_int_distribution<int> d(p, r); std::swap(A[d(rng)], A[r]); // 隨機挑 pivot 換到尾端 int q = partition(A, p, r); randomizedQuicksort(A, p, q - 1, rng); randomizedQuicksort(A, q + 1, r, rng);}

另一把引擎：Hoare partition（原始版本）習題 7-1

Lomuto 好教好證，但 Hoare 的原始 partition 實務更快（交換次數平均少三倍）： pivot 取 A[p]，兩個指標從兩端相向掃描——左指標找「≥ x」、右指標找「≤ x」，交換後繼續，交錯即停：

HOARE-PARTITION(A, p, r)
1x = A[p]　// pivot 取開頭 2i = p − 1；j = r + 1 3while true 4 repeat j = j − 1 until A[j] ≤ x 5 repeat i = i + 1 until A[i] ≥ x 6 if i < j：exchange A[i] ↔ A[j] 7 else return j
最大陷阱：Hoare 回傳的 j 不是 pivot 的最終位置！它只保證 「A[p..j] 全 ≤ x ≤ A[j+1..r] 全」——pivot 本身可能在任一側。 所以遞迴要寫 QUICKSORT(A, p, j) 與 QUICKSORT(A, j+1, r) （j 包含在左段，跟 Lomuto 的 q−1／q+1 不同）。抄錯這行是無窮遞迴的經典來源。
LomutoHoare
pivot 位置A[r]（尾）A[p]（頭）
掃描方式單向 j：p→r−1雙向相向
回傳值意義pivot 最終位置 q分界 j（pivot 不保證就位）
遞迴區間[p, q−1]、[q+1, r][p, j]、[j+1, r]
平均交換次數較多約 1/3
全相等輸入退化 Θ(n²)完美對半 Θ(n lg n)

P02 · QUICKSORT 分析

最壞 Θ(n²)、期望 Θ(n lg n)：差別全在切分平衡 §7.2、7.4

最壞情況：每次 pivot 都是最大或最小（例如已排序輸入＋固定取尾端），切分成 0 : n−1， T(n) = T(n−1) + Θ(n) = Θ(n²)——遞迴樹退化成一條鏈。

只要「成比例」就贏：即使每次都切 9 : 1，遞迴樹深度仍是 log₁₀⸝₉ n = Θ(lg n)、每層 O(n) → O(n lg n)。切分不必完美，常數比例就夠——這是 P02 recursion tree 那題不平衡樹的直接應用。

隨機化的期望分析（§7.4.2 的漂亮論證）：令 z₁<…<zₙ 為排序後元素， zᵢ 與 zⱼ要嘛比較一次、要嘛永不比較——只有當 zᵢ 或 zⱼ 是「區間 Zᵢⱼ = {zᵢ,…,zⱼ} 中第一個被選到的 pivot」才會比較：

期望比較數 O(n lg n) 完整推導（§7.4.2）
① 設變數：Xij = I{zᵢ 與 zⱼ 曾被比較}，總比較數 X = Σi<j Xij。
② 算機率：zᵢ、zⱼ 被比較 ⟺ 兩者之一是 Zij = {zᵢ, zᵢ₊₁, …, zⱼ} 中第一個被選中的 pivot （若先選中中間的某個 zk，i 與 j 被分到兩側、永不相見）。 隨機 pivot 下 Zij 的 j−i+1 個元素等可能率先當選：
Pr{Xij = 1} = 2／(j − i + 1) ③ 加總（令 k = j − i）：
E[X] = Σi=1n−1 Σj=i+1n 2/(j−i+1) = Σi=1n−1 Σk=1n−i 2/(k+1) < Σi=1n−1 2·Hn = O(n lg n)。∎
調和級數 Hn = ln n + O(1) 又出現了——與 hiring problem 同一條級數，同一套 indicator 工具。
為什麼實務最愛 quicksort
▸ 就地（O(lg n) 遞迴堆疊）、常數小、cache 友善。
▸ 隨機化後沒有「天生最壞輸入」。
▸ std::sort 的 introsort = quicksort ＋ 深度超標時切 heapsort ＋ 小區間 insertion sort：三章的知識拼在同一支函式裡。

P03 · 下界

比較排序的天花板：Ω(n lg n) §8.1

merge sort 和 heapsort 都是 Θ(n lg n)——還能更快嗎？如果演算法只透過元素兩兩比較取得順序資訊，答案是不能：

定理 8.1（決策樹論證）
把比較排序的執行過程畫成決策樹（decision tree）：內部節點是「aᵢ ≤ aⱼ？」，葉子是輸出的排列。 正確的演算法必須能產生全部 n! 種排列 → 葉子 ≥ n!。 高度 h 的二元樹葉子 ≤ 2ʰ，所以 2ʰ ≥ n! → h ≥ lg(n!) = Ω(n lg n)（Stirling）。 最壞情況的比較次數 = 根到最深葉的路徑長 ≥ Ω(n lg n)。∎

這是本課程第一個問題本身的下界——不是「還沒想到更快的方法」，而是「這個模型裡不存在更快的方法」。 merge sort／heapsort 因此是漸近最優的比較排序。想更快只有一條路：換模型——不比較，改用元素的「數值本身」當索引，這就是下一節。

QUIZ P03 · 下界的意義

counting sort 是 Θ(n + k)，k = O(n) 時線性——它違反定理 8.1 嗎？

(A) 違反，所以下界證明有漏洞
(B) 不違反：它不在「比較排序」模型裡

P04 · 線性時間排序

counting → radix → bucket：不比較，用「數的」 §8.2–8.4

counting sort：值域 [0, k] 的整數。C[v] 先數「值 = v 有幾個」，前綴和後 C[v] =「≤ v 的個數」—— 這個數字就是 v 該站的位置。從後往前放進輸出陣列 B，相等元素保持原相對順序（穩定）。按單步看放置過程：

A（輸入）
C（累計 ≤ v 個數）
B（輸出）
▸CLRS 圖 8.2 的例子：A = ⟨6,0,2,0,1,3,4,6,1,3,2⟩、k = 6。按「單步」從尾端開始放。
COUNTING-SORT(A, n, k)
1C[0:k] 全設 0 2for j = 1 to n：C[A[j]] += 1 3for v = 1 to k：C[v] += C[v−1] 4for j = n downto 1 5 B[C[A[j]]] = A[j] 6 C[A[j]] −= 1
std::vector<int> countingSort( const std::vector<int>& A, int k) { std::vector<int> C(k + 1, 0), B(A.size()); for (int a : A) ++C[a]; for (int v = 1; v <= k; ++v) C[v] += C[v - 1]; for (int i = A.size() - 1; i >= 0; --i) { B[C[A[i]] - 1] = A[i]; // 從後往前 → 穩定 --C[A[i]]; } return B;}
輸出
0 0 1 1 2 2 3 3 4 6 6
# 改寫自官方 Chapter 8/counting_sort.py（MIT License）：值域 0..k def counting_sort(A, k): C = [0] * (k + 1) for a in A: # ① 計數 C[a] += 1 for v in range(1, k + 1): # ② 前綴和：C[v] =「≤ v 的個數」 C[v] += C[v - 1] B = [None] * len(A) for a in reversed(A): # ③ 從後往前 → 穩定 B[C[a] - 1] = a C[a] -= 1 return B A = [6, 0, 2, 0, 1, 3, 4, 6, 1, 3, 2] print(counting_sort(A, 6))

radix sort：逐位排、靠穩定性接力 §8.3

RADIX-SORT(A, n, d)
1for i = 1 to d　// 從最低位到最高位 2 用穩定排序依第 i 位排序 A 3　　// 通常用 counting sort：每輪 Θ(n + k)
BUCKET-SORT(A, n) // A[i] ∈ [0, 1)
1建 n 個空桶 B[0..n−1] 2for i = 1 to n：把 A[i] 丟進 B[⌊n·A[i]⌋] 3for 每個桶：insertion sort 排桶內 4依序串接 B[0], B[1], …, B[n−1]

radix sort 的正確性靠歸納：排完第 i 位時，「後 i 位」整體有序—— 第 i 位不同者由本輪排序決定；第 i 位相同者靠穩定性保住前 i−1 輪建立的順序。 成本 Θ(d(n+k))；b 位元整數取 r 位元一組時 Θ((b/r)(n+2ʳ))，r ≈ lg n 最優。

bucket sort 的期望線性又是機率分析：桶內元素數 nᵢ 的期望 E[nᵢ²] = 2 − 1/n（indicator 變數算兩兩同桶機率）， 各桶 insertion sort 的期望總成本 Σ O(E[nᵢ²]) = O(n)。均勻分布假設是命門——全部擠進同一桶就退化 Θ(n²)。

實跑：radix sort 逐輪（CLRS 圖 8.3）＋ bucket sort（圖 8.4）— src/sorting/radix_bucket.cpp
void countingByDigit(std::vector<int>& A, int exp) { // 依第 exp 位穩定排序 std::vector<int> C(10, 0), B(A.size()); for (int a : A) ++C[(a / exp) % 10]; for (int v = 1; v < 10; ++v) C[v] += C[v - 1]; for (int i = (int)A.size() - 1; i >= 0; --i) // 從後往前 → 穩定 B[--C[(A[i] / exp) % 10]] = A[i]; A = B; }

P05 · ORDER STATISTICS

randomized select：找第 i 小，期望 Θ(n) §9.1–9.2

找中位數（或第 i 小）要排序嗎？不用。RANDOMIZED-SELECT 借 quicksort 的 partition， 但每次只遞迴一邊：pivot 落點 q 把問題切成兩半，答案只會在其中一半。 期望分析：pivot 有一半機率落在「中間 50%」，每兩輪期望把問題砍到 3/4 → 幾何級數 n + 3n/4 + 9n/16 + … = Θ(n)。

對照：quicksort 兩邊都遞迴 → Θ(n lg n)；select 只走一邊 → Θ(n)。 同一個 partition，遞迴結構決定級別。另外 §9.3 的 SELECT（中位數之中位數、5 個一組）把「期望線性」升級成最壞情況線性——保證 pivot 至少甩掉 30% 的元素，代價是常數較大，實務反而少用。
RANDOMIZED-SELECT(A, p, r, i)　// 第 i 小
1if p == r：return A[p] 2q = RANDOMIZED-PARTITION(A, p, r) 3k = q − p + 1　// pivot 在本段排第 k 小 4if i == k：return A[q] 5elseif i < k：return RANDOMIZED-SELECT(A, p, q−1, i) 6else：return RANDOMIZED-SELECT(A, q+1, r, i−k)
第 6 行的 i − k：進入右半段後「目標排名」要扣掉左半段（含 pivot）的 k 個元素。
int randomizedSelect(std::vector<int>& A, int p, int r, int i, std::mt19937& rng) { if (p == r) return A[p]; std::uniform_int_distribution<int> d(p, r); std::swap(A[d(rng)], A[r]); int q = partition(A, p, r); int k = q - p + 1; if (i == k) return A[q]; if (i < k) return randomizedSelect(A, p, q - 1, i, rng); return randomizedSelect(A, q + 1, r, i - k, rng);}

# 改寫自官方 Chapter 9/randomized_select.py（MIT License）：含 partition 全文 import random random.seed(42) def partition(A, p, r): x = A[r] i = p - 1 for j in range(p, r): if A[j] <= x: i += 1 A[i], A[j] = A[j], A[i] A[i + 1], A[r] = A[r], A[i + 1] return i + 1 def randomized_partition(A, p, r): i = random.randint(p, r) A[i], A[r] = A[r], A[i] return partition(A, p, r) def randomized_select(A, p, r, i): if p == r: return A[p] q = randomized_partition(A, p, r) k = q - p + 1 # pivot 在本段排第 k 小 if i == k: return A[q] if i < k: return randomized_select(A, p, q - 1, i) return randomized_select(A, q + 1, r, i - k) # 扣掉左邊 k 個 A = [3, 2, 9, 0, 7, 5, 4, 8, 6, 1] for i in (1, 5, 10): print("第", i, "小 =", randomized_select(A[:], 0, len(A) - 1, i))

P06 · MEDIAN OF MEDIANS

SELECT：把「期望線性」升級成「保證線性」 §9.3

randomized select 的 Θ(n) 只是期望——倒楣起來還是 Θ(n²)。 §9.3 的 SELECT（中位數之中位數）不擲骰子，用五步構造一個「保證不差」的 pivot：

SELECT 五步驟 ① 把 n 個元素5 個一組分成 ⌈n/5⌉ 組。
② 每組（≤ 5 個元素）insertion sort 後取組中位數——每組 O(1)，共 O(n)。
③ 遞迴呼叫 SELECT 找這 ⌈n/5⌉ 個中位數的中位數 x。
④ 以 x 為 pivot partition 整個陣列（x 落在第 k 位）。
⑤ 和 randomized select 一樣三岔：i = k 收工、i < k 走左、i > k 走右（只走一邊）。
為什麼 x 保證不差：數人頭
至少一半的組（約 ⌈n/5⌉/2 組）的中位數 ≥ x；每個這樣的組裡至少 3 個元素 ≥ 自家中位數 ≥ x （組中位數與它上面兩個）。扣掉 x 自己那組與不滿 5 人的尾組：
≥ x 的元素至少 3(⌈⌈n/5⌉/2⌉ − 2) ≥ 3n/10 − 6 個 對稱地 ≤ x 的也至少 3n/10 − 6 個 ⟹ 第 ⑤ 步無論走哪邊， 子問題大小 ≤ 7n/10 + 6——pivot 保證甩掉約三成的元素。
遞迴式與線性證明
T(n) ≤ T(⌈n/5⌉) + T(7n/10 + 6) + O(n)　（找中位數們的中位數＋遞迴一邊＋分組/partition）。
關鍵：1/5 + 7/10 = 9/10 < 1——兩個子問題合計比原問題小，成本形成遞減幾何級數。 用 substitution 猜 T(n) ≤ cn：cn/5 + 7cn/10 + 6c + an = 9cn/10 + 6c + an ≤ cn， 取 c 夠大（如 c ≥ 20a、n ≥ 120）即成立 → T(n) = O(n) 最壞情況保證。∎
SELECT(A, i) — 概念版
1if n ≤ 5：排序後直接回傳第 i 個 25 個一組，各組取中位數 → medians 3x = SELECT(medians, ⌈|medians|/2⌉) 4以 x 分割：lo（<x）｜eq（=x）｜hi（>x） 5依 i 落點遞迴 lo 或 hi（或直接回傳 x）
課本版就地 partition 更省空間；這裡的三路分割版本邏輯相同、好讀好寫。
int select(std::vector<int> A, int i) { if (A.size() <= 5) { std::sort(A.begin(), A.end()); return A[i - 1]; } std::vector<int> medians; // ① 5 個一組取中位數 for (std::size_t j = 0; j < A.size(); j += 5) { auto e = std::min(j + 5, A.size()); std::vector<int> g(A.begin() + j, A.begin() + e); std::sort(g.begin(), g.end()); medians.push_back(g[g.size() / 2]); } int x = select(medians, (medians.size() + 1) / 2); // ② MoM std::vector<int> lo, eq, hi; // ③ 三路分割 for (int a : A) (a < x ? lo : a == x ? eq : hi).push_back(a); if (i <= (int)lo.size()) return select(lo, i); if (i <= (int)(lo.size() + eq.size())) return x; return select(hi, i - lo.size() - eq.size()); }
