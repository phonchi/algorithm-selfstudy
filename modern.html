P00 · K-MEANS

k-means：兩步迭代的分群引擎 §33.1

把 n 個點分成 k 群，最小化「各點到所屬群中心的距離平方和」。精確解是 NP-hard； Lloyd 演算法用兩步迭代逼近：指派步（每點歸最近中心）⇄ 更新步（中心移到群平均）。 按單步看 8 個點、k = 2：

▸初始中心＝前兩個點（故意選很爛的初始）。按「單步」。
K-MEANS(P, k)（Lloyd）
1任選 k 個初始中心 2repeat 3 指派：每點歸最近的中心 4 更新：每個中心 = 該群平均 5until 指派不再變動
兩步都不會讓目標函數變大 ⟹ 必收斂；但收斂點是局部最佳，跟初始中心有關（實務跑多組初始取最好，k-means++ 改進初始化）。
for (int i = 0; i < n; ++i) { // 指派步 int best = 0; double bd = 1e18; for (int j = 0; j < k; ++j) { /* 最近中心 */ } assign[i] = best;}for (int j = 0; j < k; ++j) // 更新步 c[j] = 群 j 的座標平均;
執行輸出（src/modern/kmeans.cpp）
iter 1：分群 01111111，中心 (1, 1) (4.28571, 4)
iter 2：分群 00001111，中心 (1.375, 1.625) (6.375, 5.625)
iter 3：分群 00001111，中心 (1.375, 1.625) (6.375, 5.625)
指派不再變動，收斂。

輸出

P01 · GRADIENT DESCENT

梯度下降：沿著最陡的下坡走 §33.3

連續參數的最佳化引擎，訓練神經網路的心臟：x ← x − η·∇f(x)—— 往梯度反方向（最陡下坡）走一步，步伐大小由學習率 η 控制。 凸函數＋合適的 η 保證收斂到全域最小；深度學習的損失面不凸，但它依然是實務首選。

⚠ 學習率是雙面刃
右邊實測 f(x) = (x−3)² + 2 從 x₀ = −2 出發：η = 0.1 穩穩滑向 3； η = 0.9 在谷底兩側左右跳但仍收斂（|1−2η| < 1）； η = 1.1 每步反而離谷底更遠——發散。η 是 ML 實務中第一個要調的超參數。

大資料集算全梯度太貴時用隨機梯度下降（SGD）：每步只抽一小批樣本估計梯度——用雜訊換速度，Ch33.3 的重點應用。

GRADIENT-DESCENT(f, x₀, η, T)
1x = x₀ 2for t = 1 to T 3 x = x − η·∇f(x) 4return x
auto df = [](double x) { return 2 * (x - 3); };for (double eta : {0.1, 0.9, 1.1}) { double x = -2; for (int t = 1; t <= 30; ++t) x = x - eta * df(x); // x ← x − η∇f}
執行輸出（src/modern/gradient_descent.cpp）
η = 0.1：x = -2.0000 → -1.0000 → -0.2000 → 0.4400 → … → 2.9938，f(x) = 2.0000
η = 0.9：x = -2.0000 → 7.0000 → -0.2000 → 5.5600 → … → 2.9938，f(x) = 2.0000
η = 1.1：x = -2.0000 → 9.0000 → -4.2000 → 11.6400 → … → -1183.8816，f(x) = 1408689.8588
η 太小走得慢、剛好收斂快、太大在谷底兩側越盪越遠。
# 自寫簡例x = -2.0for t in range(30): x = x - 0.1 * 2 * (x - 3)
輸出
x = 2.9938，f(x) = 2.0

P02 · MULTIPLICATIVE WEIGHTS

乘法權重：聽誰的專家建議 §33.2

每天 N 位專家各給預測，你要當場決定跟誰；事後才知道對錯。 weighted-majority：每位專家一個權重（初始 1），按加權多數做決定； 某專家猜錯就把他的權重乘 1/2。

錯誤上界（不需任何統計假設！）
你的總錯誤數 ≤ 2.41·(最佳專家的錯誤數) + O(lg N)。
直覺：你每錯一次，至少一半的權重被砍半 → 總權重掉得快； 最佳專家的權重只在他錯時才掉 → 總權重又不能掉太多（≥ 最佳專家的權重）。兩邊夾出上界。 把「乘 1/2」改成「乘 (1−ε)」可把常數壓到 (2+ε)，這個框架（multiplicative weights update） 在線上學習、賽局、LP 求解裡到處出現。

關鍵詞是「線上（online）」：資料一筆筆到、當場決策不能反悔——與前面「拿到整個輸入再算」的離線世界完全不同的賽制。

P03 · METAHEURISTICS

metaheuristics 地圖：無保證但常常很好 概覽

NPC 問題太大、近似比又不夠用時，工程實務的第三條路：metaheuristic—— 通用的搜尋策略框架，不證明品質、靠設計好的「探索 vs 利用」平衡在大空間裡找好解。

方法靈感逃離局部最佳的機制
hill climbing爬山沒有——最容易卡住，當 baseline
simulated annealing金屬退火以溫度控制的機率接受「變差的移動」，逐步降溫
tabu search短期記憶禁忌清單擋住走回頭路
genetic algorithm演化族群＋交配＋突變維持多樣性
ACO（蟻群）螞蟻覓食費洛蒙蒸發遺忘舊路＋機率性探索
課程 TSP 四連作業——一條題目走完整學期 同一個 TSP，四種武器輪番上： ① 窮舉 O(n!)——n 稍大就爆炸（複雜度的體感）； ② 貪婪（最近鄰）——快但無保證（Ch15 的極限）； ③ Held–Karp DP O(2ⁿn²)——指數但遠勝窮舉（Ch14 位元 DP 實戰）； ④ ACO——大 n 的實務解（本節）。做完你就親手量過「理論 vs 實務」的每一格。

P04 · ACO × TSP

蟻群演算法：費洛蒙上的集體智慧 作業主線

每隻螞蟻用機率走一圈 TSP：選下一城的機率 ∝ τα·(1/d)β （τ = 費洛蒙、1/d = 能見度）。全部走完後：費洛蒙先蒸發（×(1−ρ)）， 再讓每隻螞蟻在走過的邊沉積 Q/成本——短導覽留味濃。好路越走越香、爛路自然淡忘。 按單步看 8 城（與上一章 TSP 同一組點）、每輪 8 隻螞蟻：

▸初始：所有邊費洛蒙 τ = 1（一樣粗）。按「單步」跑一輪（8 隻螞蟻）。
ACO-TSP（每輪）
1for 每隻螞蟻：從起點逐城移動 2 P(v) ∝ τ(u,v)α · (1/d(u,v))β 3全部 τ ← (1−ρ)·τ　// 蒸發 4for 每隻螞蟻的導覽（成本 c） 5 走過的每條邊 τ += Q/c　// 沉積
// 機率輪盤選下一城w[v] = std::pow(tau[u][v], alpha) * std::pow(1.0 / dist(u, v), beta);// 蒸發＋沉積for (auto& row : tau) for (double& t : row) t *= (1 - rho);tau[u][v] += Q / c; tau[v][u] += Q / c;

iter 5：目前最佳 = 14.7148
iter 40：目前最佳 = 14.7148
ACO 最佳導覽：cbadegfh，成本 = 14.7148（暴力最佳 14.7148）
找到的環 c→b→a→d→e→g→f→h 就是全域最佳環的旋轉＋反向——這組 8 城 ACO 五輪內直接命中最佳解。
