P00 · 漸近符號

O、Ω、Θ 的正式定義：兩個常數撐起一切 §3.1–3.2

資結課說「O(n²) 差不多就是成長不超過 n²」。現在把「差不多」換成可以上法庭的定義——每個符號都是一個函數集合：

三個定義（f、g 皆為漸近非負函數） O（上界）：f(n) ∈ O(g(n)) ⟺ 存在正常數 c、n₀，使得對所有 n ≥ n₀，0 ≤ f(n) ≤ c·g(n)。
Ω（下界）：f(n) ∈ Ω(g(n)) ⟺ 存在正常數 c、n₀，使得對所有 n ≥ n₀，0 ≤ c·g(n) ≤ f(n)。
Θ（緊界）：f(n) ∈ Θ(g(n)) ⟺ 存在正常數 c₁、c₂、n₀，使得對所有 n ≥ n₀，0 ≤ c₁·g(n) ≤ f(n) ≤ c₂·g(n)。
定理 3.1：f(n) = Θ(g(n)) ⟺ f(n) = O(g(n)) 且 f(n) = Ω(g(n))。

拿 f(n) = 2n² + 3n 練手：取 c = 3、n₀ = 3，則 n ≥ 3 時 2n² + 3n ≤ 2n² + n² = 3n² ✓，所以 f(n) = O(n²)。 找到一組能用的 (c, n₀) 就贏了——不需要最小的。 也要注意慣用語的陷阱：「insertion sort 的執行時間是 O(n²)」是對的（上界對所有輸入成立）， 但「是 Θ(n²)」只對最壞情況成立——已排序輸入只要 Θ(n)。

P01 · SUBSTITUTION

substitution method：先猜答案，再用歸納法釘死 §4.3

解遞迴式的第一把刀，兩步驟：① 猜一個解的形式；② 用數學歸納法找出常數並證明猜測成立。 以 T(n) = 2T(⌊n/2⌋) + n 為例，猜 T(n) = O(n lg n)，也就是要證 T(n) ≤ c·n lg n：

歸納步驟（假設對 ⌊n/2⌋ 成立）
T(n) ≤ 2·(c·⌊n/2⌋ lg⌊n/2⌋) + n ≤ c·n·lg(n/2) + n = c·n·lg n − c·n·lg 2 + n = c·n lg n − (c − 1)n。
只要 c ≥ 1，尾巴 −(c−1)n ≤ 0，於是 T(n) ≤ c·n lg n ✓。
經典陷阱：證 T(n) = O(n) 時寫出 T(n) ≤ 2(c⌊n/2⌋) + n = cn + n = O(n)——這是錯的！ 歸納法必須證出「T(n) ≤ cn」一字不差的形式；「≤ cn + n」不是「≤ cn」，多出來的 +n 沒有被吸收掉。 漸近符號不能在歸納中途偷換——歸納假設是含常數的精確不等式。

猜不準怎麼辦？先用下一節的 recursion tree 畫圖得到猜測，再回來用 substitution 補嚴謹證明——兩把刀本來就是搭配使用的。

P03 · MASTER THEOREM

Master Theorem：分治遞迴式的查表機 §4.5

第三把刀處理標準形 T(n) = a·T(n/b) + f(n)（a ≥ 1、b > 1）。 比較的是兩股勢力：葉子的總數 n^(log_b a)（watershed function）對上根部的工作 f(n)（driving function）——誰以「多項式差距」勝出誰就主導：

情況條件（ε > 0）解直覺
Case 1f(n) = O(n^(log_b a − ε))T(n) = Θ(n^(log_b a))葉子主導（成本逐層遞增）
Case 2（k ≥ 0）f(n) = Θ(n^(log_b a) lgᵏn)T(n) = Θ(n^(log_b a) lgᵏ⁺¹n)每層打平，共 lg n 層
Case 3f(n) = Ω(n^(log_b a + ε)) 且正則性條件T(n) = Θ(f(n))根部主導（成本逐層遞減）

4e 的 Case 2 比 3e 更強：允許 f(n) 帶 lgᵏn 因子（k ≥ 0）。Case 3 的正則性條件 a·f(n/b) ≤ c·f(n)（c < 1）確保成本真的逐層縮小。 注意 master theorem 不是萬能——f(n) 落在「差距不是多項式」的縫隙時（例如 f(n) = n^(log_b a)/lg n）三個 case 都不適用，得回去用前兩把刀。

P04 · STRASSEN

Strassen：少一次乘法，換一個新的指數 §4.1–4.2

n×n 矩陣乘法：三重迴圈 Θ(n³)。分治版把矩陣切四塊，直觀作法要 8 次 n/2 規模的乘法： T(n) = 8T(n/2) + Θ(n²)，由 Case 1 得 Θ(n^(log₂8)) = Θ(n³)——白忙一場，瓶頸在「8」。 Strassen 的魔術：用加減法拼出 7 個乘積 P₁…P₇ 就能組回四個象限：

Strassen 遞迴式 T(n) = 7T(n/2) + Θ(n²) → Case 1 → T(n) = Θ(n^(log₂7)) = Θ(n²·⁸¹)。 多的常數次加減法全被 Θ(n²) 吃掉；指數從 3 降到 2.81 靠的只是「8 變 7」。
STRASSEN(A, B, n) — 摘要
1if n == 1：C = A·B，return 2把 A、B 各切成四個 n/2 × n/2 象限 3P₁ = A₁₁·(B₁₂ − B₂₂)　　P₂ = (A₁₁ + A₁₂)·B₂₂ 4P₃ = (A₂₁ + A₂₂)·B₁₁　　P₄ = A₂₂·(B₂₁ − B₁₁) 5P₅ = (A₁₁ + A₂₂)·(B₁₁ + B₂₂) 6P₆ = (A₁₂ − A₂₂)·(B₂₁ + B₂₂)　　P₇ = (A₁₁ − A₂₁)·(B₁₁ + B₁₂) 7C₁₁ = P₅ + P₄ − P₂ + P₆　　C₁₂ = P₁ + P₂ 8C₂₁ = P₃ + P₄　　C₂₂ = P₅ + P₁ − P₃ − P₇
7 次遞迴乘法（第 3–6 行）＋ 常數次矩陣加減。每個 Pᵢ 都是 n/2 規模。

P05 · 機率分析與隨機化

hiring problem：indicator random variable 初登場 Ch 5

面試 n 位助理，每見到「比目前最佳者更好」就換人（雇用成本高）。最壞情況（一路變好）要雇 n 次； 但若候選人以隨機順序出現，期望雇幾次？這是機率分析（probabilistic analysis）的招牌例題，工具是：

指示隨機變數（indicator random variable） 令 Xᵢ = I{第 i 位被雇用} ∈ {0,1}。引理 5.1：E[Xᵢ] = Pr{事件發生}。 第 i 位被雇 ⟺ 他是前 i 位中最好的 ⟺ 機率 1/i（隨機順序下前 i 位誰是最佳等可能）。
期望雇用次數 E[X] = E[Σ Xᵢ] = Σ E[Xᵢ] = Σᵢ₌₁ⁿ 1/i = Hₙ = ln n + O(1)。 期望值線性（linearity of expectation）不需要獨立性——這是整套技術最好用的地方。

「假設輸入隨機」靠不住時，就自己製造隨機：先 RANDOMLY-PERMUTE 再跑——這就是隨機化演算法（randomized algorithm）， 期望值對任何輸入都成立，沒有哪個輸入是天生的最壞情況。下一章的 randomized quicksort 同一招。按按鈕實測：
