P00 · HASHING

chaining 與負載因子 α：期望 O(1) 的條件 §11.1–11.2

資結課的 hash table 複習 30 秒：h(k) 把鍵壓進 m 個 slot；碰撞用鏈結（chaining）或 開放定址（open addressing）處理。本章補上嚴謹的分析語言——一切圍著負載因子 α = n/m轉：

定理 11.1／11.2（simple uniform hashing 假設下）
chaining 的搜尋期望時間 = Θ(1 + α)——不論找不找得到。 直覺：期望鏈長就是 α；只要 n = O(m)（α 是常數），所有字典操作期望 O(1)。

P01 · UNIVERSAL HASHING

對手知道你的 hash 函數怎麼辦？——隨機選一個 §11.3–11.4

open addressing 複習：所有元素直接住在表格裡（α ≤ 1），碰撞就沿探測序列找下一格 （linear probing 有一次群聚問題；4e 主推的分析假設是 independent uniform permutation hashing）。 搜尋失敗的期望探測數約 1/(1−α)——α → 1 時爆炸，這就是 open addressing 必須提早 rehash 的原因。

更深的問題：任何固定的 hash 函數 h 都有天敵——對手挑 n 個全部映到同一 slot 的鍵，表格退化成鏈結串列。 解法與 quicksort 一模一樣：把隨機性放進演算法。

universal hashing（§11.3.3） 從一族函數 ℋ 中隨機挑一個 h 使用。ℋ 是 universal 的定義：對任兩個相異鍵 k ≠ l， Prh∈ℋ{h(k) = h(l)} ≤ 1/m——碰撞機率不超過「純隨機亂丟」。 在 universal 家族下，chaining 的期望性能對任何輸入成立（定理 11.3），沒有天生的壞鍵集。 實例：hab(k) = ((ak + b) mod p) mod m，隨機挑 a ∈ {1,…,p−1}、b ∈ {0,…,p−1}。

P02 · BST

BST 複習：一切成本 = O(h)，而 h 不受控 Ch 12

BST 性質：左子樹 ≤ 節點 ≤ 右子樹。SEARCH／MIN／MAX／SUCCESSOR／INSERT／DELETE 全部是 O(h)。 問題在 h：隨機插入期望 O(lg n)，但已排序的插入序列會退化成 h = n−1 的鏈——這是紅黑樹存在的理由。 按單步看 ⟨12, 5, 18, 2, 9, 15, 19, 17⟩ 逐步建樹：

▸按「單步」插入下一個鍵。
TREE-INSERT(T, z) — 摘要
1x = T.root；y = NIL 2while x ≠ NIL　// 追蹤落點 3 y = x 4 x = (z.key < x.key) ? x.left : x.right 5z.p = y；把 z 掛在 y 的左或右
void insert(int k) { Node* z = new Node(k); Node *y = nullptr, *x = root; while (x) { y = x; x = (k < x->key) ? x->left : x->right; } z->p = y; if (!y) root = z; else if (k < y->key) y->left = z; else y->right = z;}

2 5 9 12 15 17 18 19
2 5 9 15 17 18 19

TREE-DELETE 三情境（transplant 版）：① 無左子 → 右子頂上；② 無右子 → 左子頂上； ③ 兩子俱在 → 用後繼 y = MINIMUM(z.right)接手（y 必無左子），y 先脫離原位再取代 z。inorder 順序保持不變。

P03 · RED-BLACK 性質

紅黑樹：五條性質買到 h ≤ 2 lg(n+1) §13.1–13.2

紅黑樹 = BST ＋ 每節點一個顏色位，維持五條性質：

五條性質 ① 每個節點是紅或黑。
② 根是黑的。
③ 每個葉子（NIL 哨兵）是黑的。
④ 紅節點的兩個子節點都是黑的（不能紅紅相連）。
⑤ 每個節點到其所有後代葉子的路徑上，黑節點數相同（黑高度 bh 一致）。
引理 13.1
n 個內部節點的紅黑樹高度 h ≤ 2 lg(n + 1)。
直覺：性質 ⑤ 說每條根到葉路徑黑節點數相同（= bh）；性質 ④ 說紅節點不能連續 → 任何路徑長 ≤ 2·bh。最短可能全黑（bh）、最長紅黑交錯（2bh）——最長路徑不超過最短的兩倍， 「近似平衡」就夠讓一切操作 O(lg n)。

旋轉（rotation）是修樹的板手：LEFT-ROTATE(x) 讓右子 y 升上來、x 降為 y 的左子、 y 原本的左子樹 β 過繼給 x 當右子樹——O(1) 個指標操作、BST 性質不變（inorder 順序完全不動）。

LEFT-ROTATE 前後
x y / \ → / \ α y x γ / \ / \ β γ α β
α ≤ x ≤ β ≤ y ≤ γ 在旋轉前後都成立——inorder 不變是旋轉合法性的全部理由。
為什麼不用 AVL？
AVL 更矮（≈1.44 lg n）但插刪旋轉較多；紅黑樹每次插入最多 2 次旋轉、刪除最多 3 次。std::map／Linux CFS 都選紅黑樹。

P04 · INSERT FIXUP

RB-INSERT-FIXUP：三個 case 修回五性質 §13.3

插入流程：照 BST 插入 → 新節點 z 塗紅（只可能壞性質②或④，不動性質⑤）→ FIXUP 往上修。 while 迴圈的 invariant：只有「z 是紅、z.p 也是紅」這一處違規。看叔叔 y 的臉色分三個 case（z.p 是左子的版本；右子鏡像）：

Case條件動作效果
1叔叔 y 是紅父、叔塗黑，祖父塗紅；z ← 祖父違規上推兩層（可能迭代）
2叔黑、z 是內側（右）子z ← z.p 後 LEFT-ROTATE轉成 Case 3
3叔黑、z 是外側（左）子父塗黑、祖父塗紅，RIGHT-ROTATE(祖父)違規消失，迴圈結束
