P00 · AUGMENTING

擴充資料結構：舊樹掛新欄位 Ch 17 · 補充

不發明新結構，在紅黑樹的每個節點多掛一個欄位就解鎖新操作。 經典範例 order-statistic tree：掛 size（子樹節點數）→ OS-SELECT（第 i 小）與 OS-RANK（某鍵排第幾）都是 O(lg n)。

方法論（定理 17.1）
若欄位 f(x) 只需要 x、x.left、x.right 的資訊就能算出， 插入/刪除/旋轉時就能 O(1) 局部修復、整體維護仍 O(lg n)。 size = size(左) + size(右) + 1 ✓；反之「x 的全域名次」直接掛會壞——插入最小鍵時全部名次都要改。

同框架的第二個範例區間樹：掛 max（子樹內最大右端點），INTERVAL-SEARCH 找「與 [lo, hi] 重疊的區間」O(lg n)——行程衝突偵測的核心。

OS-SELECT(x, i) // 第 i 小
1r = x.left.size + 1　// x 在子樹內的名次 2if i == r：return x 3if i < r：return OS-SELECT(x.left, i) 4else：return OS-SELECT(x.right, i − r)
Node* osSelect(Node* x, int i) { int r = sz(x->l) + 1; if (i == r) return x; return i < r ? osSelect(x->l, i) : osSelect(x->r, i - r);}
執行輸出（src/advanced/os_tree.cpp，18 個鍵）
OS-SELECT(root, 10) = 21（第 10 小的鍵）
OS-RANK(38) = 15
root 的 size = 18（= 全部 18 個鍵）
示範版掛在未平衡 BST 上聚焦 size 欄位；課本掛紅黑樹保證 O(lg n)。

P01 · B-TREE

B-tree：為磁碟而生的矮胖樹 Ch 18 · 補充

記憶體的樹在乎比較次數；磁碟的樹在乎 I/O 次數——讀一個節點 = 一次磁碟讀取（毫秒級，比 CPU 慢百萬倍）。 B-tree 把節點撐大：最小分支度 t，每節點 t−1 ~ 2t−1 個鍵、t ~ 2t 個孩子，一個節點塞滿一個磁碟頁。

高度定理（定理 18.1）
n 個鍵的 B-tree 高度 h ≤ logt((n+1)/2)。 資料庫等級 t = 1000：10 億個鍵高度 ≤ 3——三次磁碟讀取碰到任何資料。 紅黑樹同樣資料高度約 30，磁碟上是 10 倍的 I/O。

插入的靈魂是 split-child：路過全滿節點（2t−1 鍵）先分裂——中位鍵上浮、剩下對半拆； 根分裂是 B-tree 唯一長高的方式（往上長，永遠全葉同深）。

B-TREE-SPLIT-CHILD(x, i)
1y = x.ci（滿：2t−1 鍵）；新節點 z 2z 接收 y 的後 t−1 個鍵（與後 t 個孩子） 3y 剩前 t−1 個鍵 4y 的中位鍵上浮插入 x；z 成為 x 的新孩子
// t = 2（2-3-4 樹），插入 F S Q K C L H T V W M R Nchar mid = y->keys[T - 1];z->keys.assign(y->keys.begin() + T, y->keys.end());y->keys.resize(T - 1);x->keys.insert(x->keys.begin() + i, mid); // 上浮

P02 · FIBONACCI HEAP

Fibonacci heap：把 decrease-key 壓到 O(1) 4e 線上章節 · 補充

binary heap 的 DECREASE-KEY 要 O(lg n)；Prim/Dijkstra 各要做 E 次——瓶頸就在這。 Fibonacci heap 用「懶惰」哲學改寫成本表（攤還分析的極致，工具全來自 Ch16）：

操作binary heap（最壞）Fibonacci heap（攤還）
INSERTO(lg n)O(1)
MINIMUMO(1)O(1)
UNION（合併兩堆）O(n)O(1)
DECREASE-KEYO(lg n)O(1)
EXTRACT-MINO(lg n)O(lg n)
機制速寫＋它買到什麼 根串列想塞就塞（insert/union 只是接串列）；EXTRACT-MIN 時才一次「整理」（同度數樹合併）， 把欠的工作攤還掉。DECREASE-KEY 直接把節點剪下丟回根串列（cut＋cascading cut），勢能付帳。
效果：Prim/Dijkstra 從 O(E lg V) 降到 O(E + V lg V)——稠密圖（E ≈ V²）時是 lg V 倍加速。 實務上常數大、指標繁複，多數場景 binary heap 就夠；它的價值一半在理論界標、一半在攤還分析教科書級示範。

P03 · LINEAR PROGRAMMING

線性規劃：最佳化的共同語言 Ch 29 · 補充

標準型：在線性限制 Ax ≤ b、x ≥ 0 之下最大化線性目標 cTx。 幾何圖像是一切直覺的來源：

三個核心事實 ① 可行域是凸多面體（半平面的交集）。
② 線性目標的最佳值若存在，必可在頂點達成——找最佳解 = 逛頂點。
③ 單體法（simplex）沿邊從頂點走向更好的鄰頂點，實務飛快（最壞指數）； 橢球法／內點法給了多項式保證——LP ∈ P。

它是「共同語言」：最大流、最短路徑、二人零和賽局都能寫成 LP； 上一章 vertex cover 的近似還能走「LP 鬆弛 → 解實數版 → 捨入」的路線（x ≥ ½ 就選，另一個 2-近似）。 整數規劃（ILP）則是 NP-hard——「整數」二字就是 P 與 NP-hard 的分界線。

P04 · STRING MATCHING

KMP：失配也不回頭 §32.4 · 補充

在長度 n 的文字裡找長度 m 的 pattern。暴力法每個位移比到底：O((n−m+1)m)。 KMP 的觀察：失配時，已匹配的前綴自己就告訴你能滑多遠—— 預算 前綴函數 π[q] =「P₁..q 的最長真前綴兼後綴長度」，失配時 q 退到 π[q]， 文字指標 i 永不回頭 → O(n + m)。按單步看 P = ababaca 掃 T = abababacaba：

▸π(ababaca) = [0 0 1 2 3 0 1]。按「單步」逐字元比對。
KMP-MATCHER(T, P) — 核心
1q = 0　// 已匹配字元數 2for i = 1 to n 3 while q > 0 and P[q+1] ≠ T[i]：q = π[q] 4 if P[q+1] == T[i]：q = q + 1 5 if q == m：回報位移 i − m；q = π[q]
攤還論證：q 每輪最多 +1、while 只會把 q 減小且 q ≥ 0 ⟹ 總下修次數 ≤ 總上修次數 ≤ n → O(n)。
int q = 0;for (int i = 0; i < n; ++i) { while (q > 0 && P[q] != T[i]) q = pi[q]; if (P[q] == T[i]) ++q; if (q == m) { /* shift = i-m+1 */ q = pi[q]; }}
執行輸出（src/advanced/kmp.cpp）
π(ababaca) = [0 0 1 2 3 0 1]
